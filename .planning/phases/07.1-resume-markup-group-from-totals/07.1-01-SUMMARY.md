## Decisions Made

- activatePreset follows the exact `activate()` structure (INITIAL_STATE spread), not a partial setState update — guarantees no stale fields from prior state bleed through
- count preset mode is `'placing'` (not `'naming'`): the naming popup is intentionally bypassed since the user selected an existing named item in the Totals Panel
- wall height uses `preset.wallHeight ?? 2400` inline spread so other non-wall tools don't see the field in the spread; consistent with Pitfall 7 guidance from RESEARCH.md
