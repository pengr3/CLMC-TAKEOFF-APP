<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Human UAT — arm from totals panel end-to-end</name>
  <what-built>
    End-to-end arm-from-totals-panel feature:
    - TotalsRow: + hover affordance (20px slot, Plus icon, tooltip "Add more to this group")
    - TotalsRow: onArmTool fires on click
    - TotalsPanel/TotalsCategoryBlock: prop threading with category.name curry
    - CanvasViewport: setChainArmedFromTotals export + _activatePresetRef module ref
    - useMarkupTool: activatePreset() skips naming popup, sets mode:placing/drawing + chainArmed:true
    - App.tsx: handleArmTool resolves toolType/name/color/wallHeight, calls setChainArmedFromTotals BEFORE setActiveTool
    - RibbonToolbar: setChainArmedFromTotals(null) in disarm branch
  </what-built>
  <how-to-verify>
    Prerequisites: Open the app. Load a project with at least one markup of each type (count, linear, area, perimeter, and if possible wall). The totals panel must show populated rows.

    **Scenario A — Count row arms count tool (SC-1, SC-2):**
    1. In the totals panel, hover over a count item row. Confirm: a "+" icon appears at the right end of the row. Mouse off — icon disappears.
    2. Click the count item row. Confirm: (a) Count tool becomes active in the ribbon (blue underline), (b) ChainBadge chip showing the item name appears on the Count button, (c) canvas cursor becomes crosshair. No naming popup appeared.
    3. Click on the canvas to place a count pin. Confirm: pin appears with the pre-armed name/color, no popup shown.
    Expected: PASS if count pin placed without popup.

    **Scenario B — Linear/Area/Perimeter row arms matching tool (SC-2):**
    1. Click a linear item row. Confirm: Linear tool active, ChainBadge shows, cursor crosshair, ready to draw.
    2. Click an area item row. Confirm: Area tool active, ChainBadge shows.
    3. Click a perimeter item row. Confirm: Perimeter tool active, ChainBadge shows.
    Expected: PASS if each tool activates without popup, matching the row's item type.

    **Scenario C — + icon affordance visible (SC-3):**
    Hover each row type. Confirm: "+" icon appears only on hover, disappears on mouse leave. Icon is to the right of the UoM column. Icon does not cause the row to reflowed/layout-jump.
    Expected: PASS if icon appears/disappears cleanly on all row types.

    **Scenario D — Chain mode continues after arming (SC-4):**
    1. Click a count item row in the totals panel to arm.
    2. Click canvas twice to place two pins. Confirm: both placements use the same name/color, no popup between them.
    3. Click a linear item row to arm.
    4. Draw two separate linear markups back-to-back. Confirm: chain mode continues without re-prompting.
    Expected: PASS if successive placements require no popup.

    **Scenario E — Esc disarms (SC-5):**
    1. Arm any tool by clicking a totals row.
    2. Press Esc on the keyboard.
    3. Confirm: (a) ChainBadge disappears from the tool button, (b) canvas cursor reverts to default/pointer, (c) active tool deselects.
    Expected: PASS if all three visual changes occur on Esc.

    **Scenario F — Re-click tool button disarms (SC-5):**
    1. Arm any tool by clicking a totals row.
    2. Click the same tool button in the ribbon again.
    3. Confirm: (a) ChainBadge disappears, (b) tool deactivates (no longer highlighted in ribbon).
    Expected: PASS if tool disarms on second click.

    **Scenario G — Wall row arms with height inheritance (if wall markups exist):**
    1. Ensure at least one wall markup exists with height != 2400mm (e.g., 3000mm).
    2. Click the corresponding wall item row in the totals panel.
    3. Draw a wall segment. At commit time (the popup that appears), confirm the wall height field pre-fills with 3000mm (the inherited height), not 2400mm.
    Expected: PASS if wall height inherited. (Skip if no wall markups in project — mark N/A.)
  </how-to-verify>
  <resume-signal>Type "approved" if all applicable scenarios PASS. Or describe which scenarios failed and what happened.</resume-signal>
