## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EditMarkupCommand → undoStack | EditMarkupCommand always routed through pushCommand — no direct set() on markup fields that bypasses undo stack |
| onEdit callback → CanvasViewport | onEdit receives no untrusted input — it signals intent; CanvasViewport reads markup state from the store, not from the callback payload |
| MarkupNamePopup onConfirm → editMarkup | The onConfirm payload { name, categoryName, color } flows through D-13 canonical substitution in handleConfirm before reaching editMarkup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Tampering | EditMarkupCommand bypassing undo stack | mitigate | editMarkup action uses pushCommand(s.undoStack, cmd) inside set() — same pattern as placeMarkup/deleteMarkup/recolorGroup; no direct mutation of markup fields outside this path |
| T-07-03-02 | Tampering | D-13 canonical substitution bypassed via edit path | mitigate | handleConfirm (Wave 2) runs substitution before onConfirm for ALL modes including 'edit'; the editMarkup action receives the already-canonicalized categoryName |
| T-07-03-03 | Tampering | MarkupCommand undo/redo fallthrough (Landmine 1) | mitigate | Explicit if (cmd.type === 'edit-markup') branch with early return in both undo() and redo() — added BEFORE the cmd.markup.page fallthrough line; TypeScript narrows cmd to place/delete union for the fallthrough |
| T-07-03-04 | Tampering | Wrong popup position due to viewport vs container coords (Landmine 4) | mitigate | CanvasViewport subtracts containerRef.current.getBoundingClientRect().left/top from contextMenu.x/y before setting editPopup state; the popup receives container-relative coords matching its position:absolute anchor |
| T-07-03-05 | Tampering | XSS via markup name rendered in edit popup | accept | React JSX renders name as text node via textContent/value; no dangerouslySetInnerHTML used; React 19 default escaping applies |
