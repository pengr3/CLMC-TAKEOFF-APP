  <how-to-verify>
    Run `npm run dev` to start the app. Open a PDF that has at least one page with markups or place some markups during testing.

    **Scenario A — Canvas fills the window (UI-01):**
    1. Open a PDF. Confirm the canvas fills the full center column — no dark gutter visible on the right or bottom when the page is at fit-to-window zoom.
    2. Resize the window (drag the corner). Confirm the Stage tracks the new dimensions immediately.
    3. Drag the right-side panel Splitter. Confirm canvas resizes as the panel width changes.
    Expected: No 800×600 Stage lock. No gap between canvas edge and window frame.

    **Scenario B — TotalsPanel cleanup (VIEW-01):**
    1. Place several markups on a calibrated page (include linear/area types with different categories).
    2. Open the TotalsPanel (right panel).
    3. Confirm: no "Total" bar at the bottom of the panel. No subtotal rows within each category section.
    4. Confirm: category headings, individual item rows, collapse/expand, right-click context menu all still work.
    Expected: Clean panel — only category headings and item rows visible.

    **Scenario C — CalibrationDialog dropdown (UI-01):**
    1. Click the "Set Scale" toolbar button or click "Set scale" in the canvas header bar.
    2. Draw a scale line on the canvas.
    3. In the CalibrationDialog, click the unit dropdown.
    4. Confirm: all 5 options (m, ft, mm, cm, in) are visible and clickable. The dropdown list is NOT clipped or hidden behind the modal backdrop.
    5. Confirm: the secondary button reads "Discard Scale" (not "Cancel").
    Expected: Dropdown fully visible. Correct button label.

    **Scenario D — Category dedup keyboard nav (MARK-03):**
    1. Start placing a count markup (click the Count tool, then click on the plan).
    2. In the MarkupNamePopup, type a category name that partially matches an existing category (e.g., if 'Electrical' exists, type 'elec').
    3. Confirm: suggestion list appears.
    4. Press ArrowDown — confirm a row in the list gets highlighted (hoverSurface background + accent left border).
    5. Press ArrowDown again — confirm highlight moves to next row.
    6. Press ArrowUp — confirm highlight retreats.
    7. Press Enter — confirm the highlighted category name is populated in the input field. Confirm the suggestion list closes. Confirm the popup is NOT submitted (name field is still editable).
    Expected: Keyboard navigation works. Enter selects suggestion without submitting popup.

    **Scenario E — Category canonical substitution (MARK-03):**
    1. Place a markup in category 'Electrical' (exact casing). 
    2. Place a second markup — type 'electrical' (all lowercase) in the category field. Click Save/Start Count WITHOUT using the autocomplete.
    3. Open the TotalsPanel. Confirm both markups appear under the SAME category 'Electrical' (not a new 'electrical' category).
    Expected: D-13 substitution prevents typo-duplicate categories.

    **Scenario F — Post-commit markup editing (MARK-03):**
    1. Place a count markup with name='Switch', category='Electrical', some color.
    2. Right-click the markup pin. Confirm: the context menu shows 'Edit' as the FIRST item (before 'Recolor group' and 'Delete').
    3. Click 'Edit'. Confirm: MarkupNamePopup opens pre-filled with 'Switch', 'Electrical', and the current color selected.
    4. Change the name to 'Outlet', change the category to 'Mechanical', change the color to a different swatch.
    5. Click 'Save Changes'. Confirm:
       a. The pin on the canvas reflects the new color immediately.
       b. The TotalsPanel shows 'Outlet' under 'Mechanical' (not 'Switch' under 'Electrical').
    6. Press Ctrl+Z. Confirm ALL THREE fields revert simultaneously: name back to 'Switch', category back to 'Electrical', color back to original — in one undo step.
    7. Press Ctrl+Y. Confirm re-application: name becomes 'Outlet' again, category 'Mechanical', new color restored.
    Expected: Per-markup edit with full undo/redo symmetry.

    Type "approved" if all six scenarios pass, or describe which scenario failed and what was observed.
