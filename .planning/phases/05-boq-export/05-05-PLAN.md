<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard event → useKeyboardShortcuts → handlers.exportBoq | window-level keydown listener; isTextInputActive guard prevents firing while typing in markup-name popups |
| Toolbar onClick → onExportClick → App.tsx handleExportClick → useExport().exportBoq | Plain prop callback chain; no boundary |
| ExportResult routing in handleExportClick → setState (toast / modal / error) | In-process UI state; React handles |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05-01 | Tampering | Ctrl+Shift+E fires while user types 'E' in MarkupNamePopup, deleting their typing OR triggering an unwanted export | mitigate | `if (isTextInputActive()) return` guard at the top of the Ctrl+Shift+E branch — same pattern as Ctrl+S / Ctrl+Shift+S / Ctrl+Z. Wave 0 test asserts the guard. |
| T-05-05-02 | Denial of Service | exportToast never auto-dismisses → covers UI permanently | mitigate | useEffect with `window.setTimeout(setExportToast(null), 2000)` mirroring saveToast at App.tsx:69-73; cleanup function clears the timer on unmount or new toast. |
| T-05-05-03 | Tampering | The Toolbar button's `onClick={() => { void onExportClick() }}` swallows the returned promise — if onExportClick throws an unhandled async error, the user sees nothing | mitigate | `handleExportClick` is wrapped in try/catch INSIDE useExport's hook (Plan 04). Defensive: if the hook ever does throw, the void Promise means React warns; acceptable v1 noise. Improve in v2 with a global error boundary. |
| T-05-05-04 | Information Disclosure | exportError modal displays raw OS error which may include user's full file path | accept | Same as Phase 4.1 SaveCloseModal / OpenErrorModal — user just chose this path in the OS dialog moments ago; not a privacy regression. |
| T-05-05-05 | Spoofing | A malicious extension or DevTools eval triggers `window.api.writeBoqXlsx` directly bypassing the dialog | accept | This is the existing Electron preload-bridge trust model: whatever code runs in the renderer has the api object. v1 has no privilege boundary inside the renderer. |
</threat_model>
