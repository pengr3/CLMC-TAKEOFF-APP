## Test 1 — End-to-end save & reopen (PERS-01 + PERS-02)

**Goal:** A v2 .clmc file written by the app and re-opened later restores every detail of the project state.

**Steps:**
1. Open a real PDF: Toolbar -> Open -> pick `your-test.pdf`
2. Set scale on page 1 (draw calibration line, enter known distance)
3. Place at least 5 markups across at least 2 pages: 1 count, 1 linear, 1 area, 1 perimeter, 1 more count
4. Press Ctrl+S -> Save dialog -> choose path `your-test.clmc` -> click Save
5. Observe title bar shows "Saving..." (briefly), Save/Save As/Replace buttons disabled
6. After save: dirty asterisk gone, save toast appears "Saved"
7. Close the window (X button) — should close immediately (no dirty modal)
8. Re-launch the app
9. Toolbar -> Open -> pick `your-test.clmc`
10. Verify EVERY markup appears on the SAME page at the SAME position with the SAME name + color
11. Verify per-page scale is preserved (StatusBar shows the calibrated 1:N ratio)
12. Verify zoom + pan state restored on the page that was current at save time
13. Verify dirty asterisk is NOT shown after open

**Pass criteria:**
- [ ] Save completed without console errors
- [ ] "Saving..." indicator was visible (even briefly)
- [ ] Save/Save As/Replace buttons were disabled during save
- [ ] All markups restored to exact positions
- [ ] Per-page scales restored correctly
- [ ] Title bar after open: `your-test.clmc — CLMC Takeoff` (no asterisk)

## Test 3 — Replace Plan PDF (D-08, D-09, D-10) + DimensionMismatchModal scale warning

**Goal:** The Replace Plan PDF button swaps the embedded PDF while preserving markups; page-count validation prevents catastrophe; **dimension-mismatch modal warns about scale calibration** (review concern).

**Steps:**
1. With the project from Test 1 open: Toolbar -> Replace Plan PDF
2. **Smoke test (same file):** pick the SAME PDF used originally -> expect: project markups remain visible, dirty asterisk appears, no modal
3. Press Ctrl+S -> save -> confirm new sha256 written (overwrite same .clmc)
4. **Page-count abort test:** Toolbar -> Replace Plan PDF -> pick a PDF with a DIFFERENT page count -> expect modal "Wrong PDF — Selected PDF has X pages, but project expects Y" with [Pick again] / [Cancel]
5. Click [Cancel] — verify project state unchanged
6. **Dimension warn test (optional):** find a PDF with same page count but different page sizes; click Replace Plan PDF -> pick it -> expect modal "PDF dimensions changed — markup positions may look wrong"
7. **CRITICAL — review-concern check:** verify the modal body ALSO includes a sentence about scale calibration (e.g. "your scale calibration may need to be re-verified or reset"). This is the MEDIUM consensus review concern fix.
8. Click [Cancel] — verify state unchanged
9. (Optional) Click [Open anyway] on the dimension warning — verify markups appear (possibly drifting)

**Pass criteria:**
- [ ] Smoke test (same file) succeeds without modals; markups preserved
- [ ] Save after smoke test successful (sha256 updated)
- [ ] Page-count abort modal appears with correct numbers
- [ ] Cancel from page-count modal leaves project state unchanged
- [ ] [Optional] Dimension warning modal appears for size mismatch
- [ ] **Dimension warning modal contains scale-calibration warning text** (review concern verification)
- [ ] [Optional] Cancel from dimension modal leaves state unchanged

## Test 5 — v1 silent migration (D-06) + first-save-after-open verification (HIGH review concern)

**Goal:** A Phase 04 plain-JSON .clmc file opens with silent migration; user re-saves to get v2. **Critically: the first save AFTER opening a v1 file must succeed without "no pdfBytes" error.** This is the end-to-end verification of the HIGH consensus review concern.

**Skip condition:** if no v1 .clmc file exists on disk, mark this test "skipped" and proceed.

**Steps:**
1. Locate a Phase 04 .clmc (plain JSON, before this phase's deploy). If unavailable, this test is skipped.
2. Confirm the file is plain JSON: in DevTools console:
   ```
   node -e "console.log(require('fs').readFileSync('PATH', 'utf-8').slice(0, 50))"
   ```
   should show `{"formatVersion":1,...`
3. Toolbar -> Open -> pick the v1 .clmc
4. Verify NO migration prompt appears
5. Verify project loads with all markups + scale visible
6. Verify title bar shows the filename WITH a dirty asterisk: `name.clmc * — CLMC Takeoff`
7. **CRITICAL — HIGH review-concern check:** Press Ctrl+S -> save dialog -> choose path -> click Save
8. Verify save COMPLETES SUCCESSFULLY (no error toast, no console error)
9. After save: dirty asterisk gone, file overwritten in v2 format
10. Verify file is now ZIP: in console:
    ```
    node -e "console.log(require('fs').readFileSync('PATH').slice(0, 4).toString('hex'))"
    ```
    should print `504b0304`
11. Close app, reopen the now-v2 file -> verify markups + scale identical to before

**Pass criteria:**
- [ ] Open succeeded without prompt
- [ ] All markups + scale loaded
- [ ] Title bar showed dirty asterisk after open
- [ ] **First Ctrl+S after v1 open SUCCEEDED** (HIGH review concern verification — without explicit pdfBytes population in openV1Silent, this would fail with "no pdfBytes")
- [ ] After Ctrl+S the file became a ZIP archive (hex starts 504b0304)
- [ ] Reopening the now-v2 file shows identical state to before save
