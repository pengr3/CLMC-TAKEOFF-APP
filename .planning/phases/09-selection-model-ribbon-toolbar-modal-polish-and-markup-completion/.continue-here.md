# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure or user feedback.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: LMB-pan-during-markup — The `activeTool !== 'select'` term in the dragButtons formula was a design error; it enables LMB pan for ALL markup tools, breaking markup placement. The correct formula is `spaceHeld ? [0, 1] : [1]` unconditionally.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| activeTool-gated dragButtons | Adding `activeTool !== 'select'` to dragButtons enables LMB pan for all markup tools, overriding intent | blocking | Always use `spaceHeld ? [0, 1] : [1]` — spacebar is the only LMB-pan gate |
| undo() without selection restore | Calling undo() after a delete clears selectedMarkupIds; undo brings markups back but rings don't reappear | advisory | Peek undoStack before undo(); restore ids with setSelectedMarkupIds() after if command was delete/delete-group |

**Fix 1 — Ctrl+Z does not restore selection rings after group delete (major)**
- File: `src/renderer/src/hooks/useKeyboardShortcuts.ts` lines 92–99
- Root cause: Ctrl+Z handler calls `undo()` which restores markups to pageMarkups but never restores `selectedMarkupIds` (cleared by Delete handler)
- Fix: Before calling `undo()`, peek `useMarkupStore.getState().undoStack.at(-1)`. If type `'delete'` extract `[cmd.markup.id]`; if type `'delete-group'` extract `cmd.markups.map(m => m.id)`. After `undo()`, call `useViewerStore.getState().setSelectedMarkupIds(restoredIds)`.

**Fix 2 — LMB drag pans canvas during markup tool mode (major)**
- File: `src/renderer/src/hooks/useViewportControls.ts` line 84
- Root cause: Formula `spaceHeld || activeTool !== 'select' ? [0, 1] : [1]` sets dragButtons=[0,1] for all markup tools
- Fix: Change to `spaceHeld ? [0, 1] : [1]`. Also remove `activeTool` selector (line ~73) and `activeTool` from the useEffect deps array (line ~85) since it's no longer needed.

**Re-confirm (after fixes)**
- Re-test: Test 9 (Ctrl+A → Delete → Ctrl+Z) and Test 11 note (LMB markup drag)
</remaining_work>

<decisions_made>

- LMB pan must be suppressed during ALL tool modes — only MMB and Spacebar+LMB pan, ever
- selectedMarkupIds restoration after undo belongs in the keyboard handler (not markupStore) to preserve the architectural constraint: markupStore must not import viewerStore
- Both fixes are independent and can be done in a single /gsd-quick task
