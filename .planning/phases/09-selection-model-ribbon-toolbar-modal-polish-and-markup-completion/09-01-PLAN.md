    Create src/renderer/src/hooks/useDraggable.ts:

    Imports: useCallback, useEffect, useRef, useState from 'react'. Import React for PointerEvent type.

    Interface definitions:
    - DragPosition: { x: number; y: number }
    - UseDraggableReturn: { position: DragPosition | null; onPointerDown: (e: React.PointerEvent) => void }

    Implementation:
    - useState(null) for position (type: DragPosition | null)
    - useRef(null) for dragStart (type: { mouseX: number; mouseY: number; posX: number; posY: number } | null)
    - onPointerDown via useCallback:
        1. Check (e.target as HTMLElement).tagName — if it is INPUT, BUTTON, SELECT, or TEXTAREA: return early (D-12 interactive control guard)
        2. e.currentTarget.setPointerCapture(e.pointerId) — browser-native pointer capture (RESEARCH.md "Don't Hand-Roll")
        3. const pos = position ?? { x: 0, y: 0 }
        4. dragStart.current = { mouseX: e.clientX, mouseY: e.clientY, posX: pos.x, posY: pos.y }
      Dependencies: [position]
    - useEffect (registers window listeners):
        const onMove = (e: PointerEvent): void => {
          if (!dragStart.current) return
          const dx = e.clientX - dragStart.current.mouseX
          const dy = e.clientY - dragStart.current.mouseY
          setPosition({ x: dragStart.current.posX + dx, y: dragStart.current.posY + dy })
        }
        const onUp = (): void => { dragStart.current = null }
        window.addEventListener('pointermove', onMove)
        window.addEventListener('pointerup', onUp)
        return () => {
          window.removeEventListener('pointermove', onMove)
          window.removeEventListener('pointerup', onUp)
        }
      Dependencies: [] (listeners registered once; setPosition is stable)
    - Return: { position, onPointerDown }

    Create src/tests/use-draggable.test.ts with 5 RED test stubs, then make them pass by implementing the hook correctly.

    Pattern: position === null means "use CSS centering." The hook does NOT reset position when isOpen changes — instead, position resets automatically because the hook is called inside the modal component and when the modal unmounts+remounts (closed→opened), useState reinitialises to null. This satisfies D-14 without any additional mechanism.
