# Phase 3 — UI Design Contract

> Visual and interaction contract for the Markup Tools and Editing phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Project context:** CLMC Takeoff is a Windows desktop Electron app that already has a locked-in dark-theme design system from Phases 1–2 (`src/renderer/src/lib/constants.ts` → `COLORS`, `LAYOUT`). This phase extends — never replaces — that system. All new UI chrome (toolbar tool buttons, markup name popup, category auto-complete) must re-use the existing tokens. All canvas overlay visuals follow the zoom-compensation pattern from Phase 2 calibration (`ScalePopup`, calibration line).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual, project-owned tokens) |
| Preset | not applicable — shadcn not initialized (Electron renderer uses inline styles per Phase 2 pattern) |
| Component library | none — plain React components with inline styles (established Phase 1 convention) |
| Icon library | `lucide-react` 1.6.0 (already used in Toolbar, EmptyState) |
| Font | Inter via `@fontsource/inter` 5.2.8 (already bundled) |
| Token source | `src/renderer/src/lib/constants.ts` — `COLORS` + `LAYOUT` constants (locked) |

**Why no shadcn:** The canvas-first nature of this app (Konva stage with zoom-compensated overlays + inline popups that must screen-position against Konva coordinates) makes shadcn's Radix-based primitives a poor fit. The Phase 2 inline-style pattern is intentional and documented in `03-CONTEXT.md` under "Established Patterns: Inline styles only — no Tailwind or CSS modules on canvas-adjacent components". This UI-SPEC locks that decision rather than re-opening it.

## Spacing Scale

Declared values (all multiples of 4; follows 8-point scale already used across `Toolbar.tsx`, `StatusBar.tsx`, `ScalePopup.tsx`, `CalibrationDialog.tsx`):

| Token | Value | Usage in Phase 3 |
|-------|-------|------------------|
| xs    | 4px   | Gap between icon and label inside tool buttons; input internal vertical padding |
| sm    | 8px   | Gap between distance input and unit dropdown; spacing between stacked popup sections; action-row button gap |
| md    | 16px  | Popup outer padding (matches `ScalePopup`); toolbar horizontal padding; confirmation-toast horizontal padding |
| lg    | 24px  | Reserved — not used in this phase |
| xl    | 32px  | Reserved — not used in this phase |
| 2xl   | 48px  | Empty state hero icon size only (does not increase in this phase) |

**Layout constants (locked from Phase 1, do not modify):**
- Toolbar height: **40px** (`LAYOUT.toolbarHeight`)
- StatusBar height: **28px** (`LAYOUT.statusBarHeight`)
- TitleBar height: **32px** (`LAYOUT.titleBarHeight`)
- Tool button height: **28px** (matches existing `IconButton` in `Toolbar.tsx`)
- Input/control height: **28px** (matches `ScalePopup` inputs)

**Exceptions:**
- **Markup name popup minimum width:** 240px (matches `ScalePopup.POPUP_MIN_WIDTH` = 220, expanded by 20 to accommodate the category auto-complete list — still on 8-grid).
- **Markup name popup max width:** 280px (matches `ScalePopup.POPUP_MAX_WIDTH` = 260 expanded to 280 — 8-grid).
- **Count pin radius (before zoom compensation):** 6px unscaled — then divided by `currentZoom` per the locked zoom-compensation pattern.
- **Polyline/polygon stroke width (before zoom compensation):** 2px unscaled → divided by `currentZoom`.
- **Canvas label font size (before zoom compensation):** 12px unscaled → divided by `currentZoom`, with a minimum canvas-pixel floor of 10px so labels never disappear at extreme zoom-out (implements D-12 "always visible at all zoom levels" + Claude's Discretion "minimum font size floor").

## Typography

Locked at **3 sizes + 2 weights** (matches existing Phase 1–2 typography; no new sizes introduced). Font family: **Inter** (body + headings); **monospace** reserved for measurement-value previews only.

| Role | Size | Weight | Line Height | Usage in Phase 3 |
|------|------|--------|-------------|------------------|
| Body      | 13px | 400 (regular)  | 1.4 | Popup helper text, category auto-complete list items, confirmation toast body |
| Label/CTA | 13px | 600 (semibold) | 1.4 | Form labels (Name, Category), button text (Save, Discard, Start), active tool button text |
| Heading   | 15px | 600 (semibold) | 1.2 | Popup title if shown (e.g. for full modal variants — not used for inline popups in this phase) |

**Canvas labels (rendered in Konva, not DOM — zoom-compensated):**
- Font family: Inter (fallback: sans-serif)
- Base font size: 12px ÷ `currentZoom`, floor 10px (canvas pixels)
- Weight: 600 (semibold) — semibold reads better against PDF background noise than regular
- Stroke/shadow: 2px white text shadow (zoom-compensated) for contrast against any PDF background color

**Measurement values inside canvas labels use the same Inter 600 treatment** (not monospace) so a label like "Wall A — 12.4 m" reads as a single phrase rather than split typography. The `ScalePopup` ratio-preview monospace is for DOM preview text only, not for canvas rendering.

## Color

Color philosophy **matches the locked Phase 1 dark theme exactly**. The 60/30/10 split is already enforced by `COLORS` constants. This phase adds **one new token group**: the markup category palette (8 colors, auto-assigned). Category colors are used for **markup geometry only** — never for UI chrome.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%)    | `#1a1a1a` (`COLORS.dominant`)         | Canvas container background, input backgrounds inside popups |
| Secondary (30%)   | `#252526` (`COLORS.secondary`)        | Toolbar, StatusBar, popup card backgrounds, context menu background |
| Accent (10%)      | `#0078d4` (`COLORS.accent`)           | See reserved-for list below |
| Accent hover      | `#1a86db` (`COLORS.accentHover`)      | Primary button hover only |
| Accent active     | `#0067b8` (`COLORS.accentActive`)     | Primary button mousedown only |
| Text primary      | `#cccccc` (`COLORS.textPrimary`)      | Body text, button labels |
| Text secondary    | `#808080` (`COLORS.textSecondary`)    | Helper text, disabled labels, inactive auto-complete items |
| Text on accent    | `#ffffff` (`COLORS.textOnAccent`)     | Text inside filled accent buttons only |
| Border            | `#3c3c3c` (`COLORS.border`)           | Popup borders, input borders, dividers |
| Hover surface     | `#2d2d30` (`COLORS.hoverSurface`)     | Ghost button hover background, context menu item hover |
| Active surface    | `#37373d` (`COLORS.activeSurface`)    | Ghost button mousedown; **active tool button background** |
| Warning (semantic)| `#e8a838` (`COLORS.warning`)          | Reserved for "Not Set" / "Not Calibrated" / error states — **NOT a category color** |

**Accent (`#0078d4`) reserved EXCLUSIVELY for:**
1. Primary CTA buttons — the "Save" button in the markup name popup and the "Start" button in the Count tool popup (filled accent fill + white text)
2. Active tool button bottom-border (2px accent underline on the pressed tool — matches `IconButton active` pattern in `Toolbar.tsx`)
3. In-progress calibration/markup drawing-line stroke color (preview segment following cursor — matches existing calibration line)
4. Zoom-not-100% indicator in toolbar (already in use; unchanged)

**Accent MUST NOT be used for:**
- Category markup colors (use palette below)
- Hover states on secondary buttons
- Dividers or borders
- Text in the popup body or category auto-complete list
- Undo/redo button states

### Destructive / Cancel Color

This phase has **no destructive actions** (markup editing by click is v2 scope per deferred list). The closest is the "Discard" button on an in-progress shape — that is a **cancel, not a destructive action** (no committed data is lost since the shape was never saved).

| Role | Value | Usage |
|------|-------|-------|
| Cancel button | transparent + `COLORS.border` outline + `COLORS.textPrimary` text | "Discard" / "Cancel" buttons in popups (matches existing `ScalePopup` "Discard line" pattern). No red needed — nothing is destroyed. |

If a future destructive primitive is added (e.g. "Delete markup" in v2), it should use `#d13438` (red — already reserved as category palette color #2 per D-11, but semantic destructive context differentiates). Flag for v2 review.

### Category Color Palette (Phase 3 new tokens)

Eight distinct colors, auto-assigned in order of first category creation (locked by D-11 in CONTEXT.md). Applied **only to markup geometry** (pin fill, polyline stroke, polygon stroke + semi-transparent fill). Never to UI chrome.

| # | Hex | Name | Notes |
|---|-----|------|-------|
| 1 | `#0078d4` | Blue      | Matches `COLORS.accent` — acceptable because category colors are on canvas, not on UI chrome, so no accent-purity conflict |
| 2 | `#d13438` | Red       | |
| 3 | `#107c10` | Green     | |
| 4 | `#ca8a04` | Amber     | **Intentionally different from `COLORS.warning` (#e8a838)** per CONTEXT.md Specific Ideas — prevents confusion with "Not Set" warning state |
| 5 | `#5c2d91` | Purple    | |
| 6 | `#008272` | Teal      | |
| 7 | `#e3008c` | Pink      | |
| 8 | `#8e562e` | Brown     | |

- **Polygon fill opacity:** 20% (hex color with alpha `33`) — enough to tint the enclosed region without obscuring the PDF plan underneath.
- **Pin / polyline / polygon-outline opacity:** 100% — full-saturation geometry on canvas.
- **Palette cycling:** If a 9th+ category is created, wrap to index 0. No UI warning.

## Copywriting Contract

All copy follows the existing Phase 1–2 voice: **terse, verb-first, no exclamations, no marketing language**. Estimators are professional users; copy is operational.

### Primary CTAs (per tool)

| Element | Copy |
|---------|------|
| Toolbar Count tool button | "Count" (icon: `lucide-react` → `MapPin` or `Circle`, 16px) |
| Toolbar Linear tool button | "Linear" (icon: `Ruler` — already imported; or `Minus` — decide in planning) |
| Toolbar Area tool button | "Area" (icon: `Square` or `Hexagon`) |
| Toolbar Perimeter tool button | "Perimeter" (icon: `Frame` or `BoxSelect`) |
| Count-tool primary CTA (popup) | "Start" |
| Linear/Area/Perimeter primary CTA (popup, after shape drawn) | "Save" |
| Shape-draw cancel button (in popup) | "Discard" |
| In-progress draw cancel (keyboard) | ESC key (no label shown; matches calibration-cancel pattern) |
| Undo button (if rendered — otherwise keyboard-only) | "Undo" with `Ctrl+Z` tooltip |
| Redo button (if rendered — otherwise keyboard-only) | "Redo" with `Ctrl+Y` tooltip |

### Empty States

Three distinct empty states are relevant in Phase 3. Copy for each:

| State | Heading | Body |
|-------|---------|------|
| No PDF loaded (existing EmptyState — do not modify) | "Open a PDF floor plan to begin" | "Drag and drop a file here, or use the Open PDF button above." |
| PDF loaded, no scale set, tool clicked | (inline warning overlay, not full-screen) "Set scale before placing markups" | "Click **Set Scale** and draw a line over a known dimension first." |
| PDF loaded, scale set, no markups yet | (no dedicated empty state — the canvas itself is the target; no placeholder needed) | — |

**Copy rationale:** The "Set scale first" warning reuses the existing `COLORS.warning` token and mirrors the StatusBar "Not Set" pattern already familiar to the user. No new pattern needed.

### Error States (inline)

| Error | Copy | Color |
|-------|------|-------|
| Markup name field empty on Save | "Enter an item name" | `COLORS.warning` inline below input |
| Polyline has only one point when double-clicked | "Add at least two points before ending" | `COLORS.warning` toast, 3-second auto-dismiss |
| Polygon has fewer than 3 vertices when closing | "Add at least three points to close the shape" | `COLORS.warning` toast, 3-second auto-dismiss |
| Category name empty on Save | (allow empty — treat as "Uncategorized" with default palette color #1 blue) — no error | n/a |

No `alert()` dialogs. All errors are inline or as the existing `ConfirmationToast` pattern (bottom-center, auto-dismiss in error mode only).

### Destructive Confirmations

**None in this phase.** Delete-by-click is v2 (PROD-03, in deferred list). Undo does not require confirmation — it is a first-class reversible primitive. No red buttons, no "Are you sure?" dialogs.

### Canvas Label Copy Format (locked by D-13)

| Markup Type | Label Format | Example |
|-------------|--------------|---------|
| Count       | `● {Name} {N}` (colored dot + name + sequential number) | `● Light Switch 3` |
| Linear      | `{Name} — {value} {unit}` (em-dash, centered on polyline midpoint) | `Wall A — 12.4 m` |
| Area        | `{Name}` line 1 / `{value} {unit}²` line 2 (centered in polygon) | `Tiling Area 1` / `38.2 m²` |
| Perimeter   | `{Name}` line 1 / `P: {perim} {unit}  A: {area} {unit}²` line 2 | `Room 2` / `P: 24.6 m  A: 38.2 m²` |

- **Number formatting:** 1 decimal place for lengths, 1 decimal for areas (e.g. `12.4 m`, `38.2 m²`). Use the display unit attached to the page's scale (already in `scaleStore.getScale(page).displayUnit`).
- **Superscript "²":** use the Unicode character `²` (U+00B2), not `^2` or `m2`. Count-pin labels never have a unit.
- **Em dash:** use `—` (U+2014), not `-` or `--`, to match the project's existing `\u2014` usage in `ScalePopup.tsx` and `StatusBar.tsx`.

## Interaction Contracts

### Tool Activation (state transitions)

Extends `ActiveTool` union in `viewerStore.ts` to include `'count' | 'linear' | 'area' | 'perimeter'` (per `03-CONTEXT.md` Integration Points). Visual state of each tool button:

| State | Background | Text color | Bottom border |
|-------|------------|------------|---------------|
| Idle (inactive) | `transparent` | `#cccccc` | 2px transparent |
| Hover | `#2d2d30` (`hoverSurface`) | `#cccccc` | 2px transparent |
| Pressed (mousedown) | `#37373d` (`activeSurface`) | `#cccccc` | 2px transparent |
| Active (tool is currently selected) | `#37373d` (`activeSurface`) | `#cccccc` | **2px `#0078d4` accent** |
| Disabled (no PDF, or no scale on page) | `transparent` | `#cccccc` @ 40% opacity | 2px transparent |

This matches the existing `IconButton` active-state contract exactly — no new visual vocabulary.

### Cursor States

| Mode | Cursor |
|------|--------|
| No tool active | default (arrow) |
| Count tool active, popup not yet dismissed | default (waiting for name input) |
| Count tool active after Start clicked | `crosshair` (placing pins) |
| Linear/Area/Perimeter tool active | `crosshair` (drawing shape) |
| Hover over existing markup | `default` (no click interaction in Phase 3 — v2 PROD-03 enables selection) |

### Keyboard Shortcuts (Phase 3 scope)

| Key | Action |
|-----|--------|
| `Ctrl+Z` | Undo last markup action (place or delete) |
| `Ctrl+Y` or `Ctrl+Shift+Z` | Redo |
| `Escape` | Cancel in-progress shape / close open popup |
| `Double-click` on canvas with Linear tool active | End polyline and open naming popup |
| `Click first point` with Area/Perimeter tool active | Close polygon and open naming popup |

Tool-switch keyboard shortcuts (e.g. `C` for count) are **out of scope** per deferred list (PROD-01, v2).

### Popup Positioning Rules

`MarkupNamePopup` reuses the `ScalePopup` screen-position clamping algorithm exactly:

- **Count tool:** popup appears centered on the toolbar tool button's bottom edge (known fixed position — no canvas coordinate needed until Start is pressed).
- **Linear tool:** popup appears 20px below the midpoint of the drawn polyline, clamped to container bounds.
- **Area / Perimeter tool:** popup appears 20px below the geometric centroid of the closed polygon, clamped to container bounds.
- **Min width 240px, max width 280px**, minHeight auto, maxHeight clamped so it never extends past the canvas viewport.

### Preview Rendering During Draw (Claude's discretion — confirmed in CONTEXT.md)

- Linear: show in-progress segments at 100% opacity + a **dashed preview segment** from last click to cursor position at 60% opacity (matches calibration line preview behavior).
- Area / Perimeter: same — rendered edges solid, cursor-following edge dashed at 60% opacity.
- Count: no preview (single click places pin immediately after naming).
