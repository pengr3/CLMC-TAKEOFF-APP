## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| gsd-sdk state/roadmap mutations | `gsd-sdk query state.*` / `roadmap.*` MUTATION verbs fail on this project's STATE.md format (discovered repeatedly). | advisory | Edit `.planning/STATE.md` and `.planning/ROADMAP.md` DIRECTLY. gsd-sdk READ queries (init.*, phase-plan-index, current-timestamp, commit) are fine. |
| Worktree-isolated executors | This harness's `Agent(isolation="worktree")` does not return the worktree path/branch/agent-id metadata the workflow's manifest-based cleanup needs → cleanup hard-blocks. | advisory | Run executors SEQUENTIALLY on the main tree (no worktree). Orchestrator owns STATE/ROADMAP writes; instruct executors NOT to touch them. |
| Stale Electron main process | Renderer changes hot-reload on `npm run dev`, but MAIN-process changes (src/main/**) do NOT — the app keeps running old main code (this is exactly why GAP-2's friendly export error looked "unfixed" until a full restart). | advisory | After any src/main change, fully restart the app (Ctrl+C the dev server + relaunch, or rebuild). |
| Per-keystroke input commit | Inline-edit numeric inputs that commit on every `input` event + a store->field seed effect clobber mid-typing decimals (CR-01, High: couldn't type `0.5`). Mocked-setter tests miss it. | advisory | Commit on blur/Enter ONLY; guard the seed effect with `document.activeElement !== el`; `stopPropagation`; test against the REAL store. See EstimateRow.tsx + DefaultMarkupControl. |
| vitest.config.ts mid-work | Changing test config mid-work breaks the parallel-executor safety convention. | advisory | Do NOT modify vitest.config.ts. Tests are `src/tests/**/*.test.ts`, jsdom via `@vitest-environment jsdom` per-file + in-memory localStorage polyfill. |
