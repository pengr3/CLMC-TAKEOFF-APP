# Plan 12-04: Wave 3b — CanvasViewport: Body Drag (Translate + Group Move)

## Goal

Wire the full drag-to-translate flow for selected markups: `markupBodyDownRef` populated by markup component `onMarkupMouseDown`, `bodyDragRef` capturing the drag snapshot in `handleStageMouseDown`, live preview via `DragPreview` during `handleStageMouseMove`, `MoveMarkupsCommand` dispatched on `handleStageMouseUp`, and `bodyDraggedRef` suppression of the Konva post-mouseup click. Group move works automatically — all selected markups move by the same delta.

## Context

This plan depends on the drag refs and `dragPreview` state introduced in Plan 12-03. It does NOT touch vertex-drag (that is Plan 12-05).

Key rules from RESEARCH.md:
- `bodyDraggedRef` pattern (Pitfall 2 + Finding 4) is identical to `rubberBandDraggedRef` — prevents click from clearing selection after drag
- Stage inverse transform is mandatory for all pointer-to-page-space conversions (locked STATE.md pattern)
- StrictMode-safe: dispatch `markupStore.moveMarkups()` OUTSIDE `setState` (PATTERNS.md StrictMode section)
- 4px threshold (D-07/D-09): suppress accidental micro-drags
- Count pins use `markup.point` not `markup.points`; the `move-markups` command normalises this as `oldPoints: [markup.point]`
- **Stale closure rule:** Inside `handleStageMouseDown` and `handleStageMouseUp`, NEVER read `pageMarkups` from the React closure. Always read from `useMarkupStore.getState().pageMarkups[currentPage]` — this is the store snapshot, not stale closure state.

```typescript
type BodyDragState = {
```

**Step B — Pass `onMarkupMouseDown` to all five markup renderers** in the Layer 1b JSX section. For each `<CountPinMarkup>`, `<LinearMarkup>`, `<AreaMarkup>`, `<PerimeterMarkup>`, `<WallMarkup>`:
```jsx

```
This sets the ref before the Stage `onMouseDown` fires (Konva bubbles child events first).

**Step C — Pass `overridePoints`/`overridePoint` from `dragPreview` to each renderer** in Layer 1b. Compute per-markup overrides at render time:

For point-array markups (`LinearMarkup`, `AreaMarkup`, `PerimeterMarkup`, `WallMarkup`):
```jsx
overridePoints={
  dragPreview?.type === 'body' && dragPreview.deltas[m.id]
    ? (m as LinearMarkupType).points.map(p => ({
        x: p.x + dragPreview.deltas[m.id].x,
        y: p.y + dragPreview.deltas[m.id].y
      }))
    : dragPreview?.type === 'vertex' && dragPreview.markupId === m.id
      ? dragPreview.points
      : undefined
}
```

For count pins:
```jsx
overridePoint={
  dragPreview?.type === 'body' && dragPreview.deltas[m.id]
    ? {
        x: (m as CountMarkup).point.x + dragPreview.deltas[m.id].x,
        y: (m as CountMarkup).point.y + dragPreview.deltas[m.id].y
      }
    : undefined
}
```

**Why:** This single task wires the `onMarkupMouseDown` upstream signal and the `overridePoints` downstream rendering. Both are prerequisites for the drag handlers in T2a/T2b.

### T2a: Implement `handleStageMouseDown` body-drag branch and `handleStageMouseMove` body-drag branch

**Type:** edit
**File:** `src/renderer/src/components/CanvasViewport.tsx`

**What:**

**Step A — `handleStageMouseDown` — add body-drag branch BEFORE the rubber-band check:**

Insert the following block AFTER the `isMarkupTool(activeTool) && !spaceHeld` markup-mode capture block, BEFORE the `if (activeTool !== 'select') return` line:

```typescript
// Phase 12 D-07: body-drag branch — fires when a selected markup was mousedown'd.
// markupBodyDownRef is set by the markup component's onMarkupMouseDown BEFORE this
// Stage-level handler fires (Konva child events bubble up first).
const bodyTargetId = markupBodyDownRef.current
markupBodyDownRef.current = null  // consume immediately — prevent double-trigger
if (
  bodyTargetId &&
  activeTool === 'select' &&
  !spaceHeld &&
  selectedMarkupIds.includes(bodyTargetId)
) {
  e.evt.stopPropagation()  // prevent rubber-band from starting
  const pt = stage.getAbsoluteTransform().copy().invert().point(pointer)
  // Snapshot start positions for all selected markups.
  // Read from store snapshot, not stale closure
  const currentPageMarkups = useMarkupStore.getState().pageMarkups[currentPage] ?? []
  const startPositions: Record<string, StagePoint | StagePoint[]> = {}
  for (const id of selectedMarkupIds) {
    const markup = currentPageMarkups.find(m => m.id === id)
    if (!markup) continue
    if (markup.type === 'count') {
      startPositions[id] = { ...markup.point }
    } else {
      startPositions[id] = [...markup.points]
    }
  }
  bodyDragRef.current = {
    ids: [...selectedMarkupIds],
    origin: pt,
    startPositions
  }
  return  // suppress rubber-band
}
```

**Step B — `handleStageMouseMove` — add body-drag preview update BEFORE the rubber-band check:**

In `handleStageMouseMove`, after the null checks for `stage` and `pointer`, BEFORE the `const rb = rubberBandRef.current` check:

```typescript
// Phase 12: body drag live preview — update delta for all dragged markups.
const bd = bodyDragRef.current
if (bd) {
  const pt = stage.getAbsoluteTransform().copy().invert().point(pointer)
  const dx = pt.x - bd.origin.x
  const dy = pt.y - bd.origin.y
```

### T2b: Implement `handleStageMouseUp` body-drag commit and `handleStageClick` suppression

**Type:** edit
**File:** `src/renderer/src/components/CanvasViewport.tsx`

**What:**

**Step C — `handleStageMouseUp` — add body-drag commit BEFORE the rubber-band check:**

In `handleStageMouseUp`, the body-drag check must go BEFORE the existing rubber-band check. Replace the function opening with:

```typescript
const handleStageMouseUp = useCallback(() => {
  // Phase 12: body drag commit — fires before rubber-band cleanup.
  const bd = bodyDragRef.current
  if (bd) {
    const stage = stageRef.current
    const pointer = stage?.getPointerPosition()
    if (stage && pointer) {
      const pt = stage.getAbsoluteTransform().copy().invert().point(pointer)
      const dx = pt.x - bd.origin.x
      const dy = pt.y - bd.origin.y
      const moved = Math.abs(dx) > 4 || Math.abs(dy) > 4  // D-09 4px threshold
      if (moved) {
        // Build the move-markups command entries.
        // Read from store snapshot, not stale closure
        const currentPageMarkups = useMarkupStore.getState().pageMarkups[currentPage] ?? []
        const moves: Array<{ markupId: string; page: number; oldPoints: StagePoint[]; newPoints: StagePoint[] }> = []
        for (const id of bd.ids) {
          const markup = currentPageMarkups.find(m => m.id === id)
          if (!markup) continue
          if (markup.type === 'count') {
            moves.push({
              markupId: id,
              page: markup.page,
              oldPoints: [markup.point],
              newPoints: [{ x: markup.point.x + dx, y: markup.point.y + dy }]
            })
          } else {
            moves.push({
              markupId: id,
              page: markup.page,
              oldPoints: [...markup.points],
              newPoints: markup.points.map(p => ({ x: p.x + dx, y: p.y + dy }))
            })
          }
        }
        // StrictMode-safe: dispatch OUTSIDE setState (PATTERNS.md).
        useMarkupStore.getState().moveMarkups(moves)
        bodyDraggedRef.current = true  // suppress Konva click (Pitfall 2, mirrors rubberBandDraggedRef)
      }
    }
    bodyDragRef.current = null
    setDragPreview(null)
    return  // done — do not fall through to rubber-band handler
  }

  // Rubber-band release (existing code follows unchanged)
  const rb = rubberBandRef.current
  if (!rb) return
  // ... existing rubber-band commit code ...
```

The existing rubber-band code remains untouched below the new block.

**Step D — `handleStageClick` — add `bodyDraggedRef` suppression at the top**, mirroring the `rubberBandDraggedRef` suppression:

After the existing `rubberBandDraggedRef` check:
```typescript
// Phase 12: body-drag suppression — mirrors rubberBandDraggedRef pattern.
if (bodyDraggedRef.current) {
  bodyDraggedRef.current = false
  return
}
```

**Why:** This implements D-07 (translate with 4px threshold), D-08 (group move via `selectedMarkupIds`), and fixes Pitfall 2 (post-drag click suppression). The `useMarkupStore.getState().moveMarkups()` call outside `setState` is the StrictMode-safe pattern established in `useMarkupTool.ts`. All `pageMarkups` reads use `useMarkupStore.getState()` (store snapshot) not the closed-over React state variable.
