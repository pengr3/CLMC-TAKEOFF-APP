# Plan 12-06: Wave 4 — UAT Checkpoint and Phase Closure

## Goal

Run the full Phase 12 UAT scenario list, fix any regressions found, update ROADMAP.md and STATE.md to mark Phase 12 complete.

## Context

This plan is the human-verify gate for Phase 12. It executes after all implementation plans (12-01 through 12-05) are complete. The UAT covers all five Phase 12 success criteria from ROADMAP.md, plus regression checks on existing Phase 9/10 features (selection model, rubber-band, undo stack).

---

## Tasks

### T1: Human UAT — Phase 12 scenario list

**Type:** verify (checkpoint:human-verify)

**What:**

Run the following UAT scenarios in the app. Report any failures.

**Setup:** Open a multi-page PDF. Set scale on page 1. Place at least one of each markup type:
- Linear with 3 points
- Area (closed polygon) with 4 points
- Perimeter with 4 points
- Wall with 3 points
- Two count pins

---

**Scenario A — Vertex edit activation (D-04):**
1. Click a linear markup — selection ring appears (no handles).
2. Click it again — 8×8px white square handles appear at each vertex; markup color border on handles.
3. Click a different part of the canvas — handles disappear.
PASS criteria: handles appear on second click only; count of handles = count of vertices.

**Scenario B — Handle zoom compensation (D-05):**
1. Enter vertex edit mode on a linear markup.
2. Zoom in to 4× — handle squares appear the same size on screen (not larger).
3. Zoom out to fit — handle squares appear the same size on screen (not smaller).
PASS criteria: handles are visually constant-size at any zoom.

**Scenario C — Enter to commit (D-06):**
1. Enter vertex edit mode. Drag a handle to a new position.
2. Press Enter — handles disappear; markup stays at new vertex position.
3. Ctrl+Z — markup returns to original vertex positions.
PASS criteria: Enter commits, Ctrl+Z undoes as a single step.

**Scenario D — Escape to restore (D-06):**
1. Enter vertex edit mode. Drag a handle to a new position.
2. Press Escape — handles disappear; markup returns to ORIGINAL vertex positions (before the drag).
PASS criteria: Escape restores original, no undo entry created.

**Scenario E — Click-outside to commit (D-06):**
1. Enter vertex edit mode. Drag a handle.
2. Click on empty canvas — handles disappear; markup stays at new position.
3. Ctrl+Z — reverts to original.
PASS criteria: Click-outside commits; single undo step.

**Scenario F — Click-outside to commit when clicking a different markup (D-06):**
1. Enter vertex edit mode on markup A. Drag a handle.
2. Click markup B — markup A's handles disappear (committed), markup B gets selection ring.
PASS criteria: vertex edit on A is committed; B is now selected.

**Scenario G — Translate selected markup (D-07):**
1. Click a linear markup to select it.
2. Drag the markup body (not on a handle) to a new position.
3. Release — markup moves to new position.
4. Ctrl+Z — markup returns to original position.
PASS criteria: body drag translates; single Ctrl+Z undo.

**Scenario H — 4px movement threshold (D-09):**
1. Select a markup. Click and release with only a tiny movement (< 4px).
2. Markup should NOT translate — the click is treated as a click, not a drag.
PASS criteria: micro-movement does not cause translation.

**Scenario I — Group move (D-08):**
1. Rubber-band select two markups.
2. Drag one — both move by the same delta.
3. Release. Ctrl+Z — BOTH return to original positions in one undo step.
PASS criteria: group move with single undo command.

**Scenario J — Count pin translate-only:**
1. Select a count pin. Click again — NO vertex handles appear.
2. Drag the count pin body — it translates. Ctrl+Z restores.
PASS criteria: count pin never enters vertex edit mode; does translate.

**Scenario K — Vertex edit with area/perimeter/wall:**
1. Enter vertex edit on an area markup (closed polygon). Drag a corner handle.
2. The polygon closes correctly (last vertex connects to first).
3. Commit. Ctrl+Z restores.
PASS criteria: area/perimeter/wall geometry edits correctly.

**Scenario L — Regression: rubber-band selection still works:**
1. With select tool active, drag across empty canvas — rubber-band rectangle appears.
2. Release — markups in the band are selected (selection ring).
PASS criteria: rubber-band is unaffected by Phase 12 changes.

**Scenario M — Regression: Ctrl+Z during in-progress drawing still pops last point:**
1. Start drawing a linear markup. Place 3 points.
2. Ctrl+Z — last point removed; still in drawing mode.
PASS criteria: Phase 10 granular undo unaffected.

**Scenario N — Vertex edit cleared on page navigation:**
1. Enter vertex edit mode on page 1.
2. Navigate to page 2 — handles disappear.
3. Navigate back to page 1 — no handles (vertex edit is gone).
PASS criteria: vertex edit is page-scoped.
