# Plan 12-05: Wave 3c — CanvasViewport: Vertex Drag Flow + Click-Outside Commit

## Goal

Complete the vertex-edit interaction loop: wire `onHandleMouseDown` on `VertexHandleOverlay` to start vertex drag in `handleStageMouseDown`, update `dragPreview` during `handleStageMouseMove`, dispatch `moveVertex` on `handleStageMouseUp`, and implement click-outside-to-commit in `handleStageClick`.

## Context

This is the final CanvasViewport wiring plan. It depends on the refs and drag-preview from 12-03 and the body-drag handlers from 12-04. After this plan, all four locked decisions (D-04 through D-09) are fully implemented.

Key points from RESEARCH.md:
- Handle mousedown detected by `e.target.name().startsWith('handle-')` in `handleStageMouseDown` — OR by the `onHandleMouseDown` callback on `VertexHandleOverlay` setting a ref (simpler, more reliable per Finding 4 Note)
- `vertexDragRef` captures: `{ markupId, vertexIndex, origin: stagePoint, originalPoints: StagePoint[] }`
- Live preview: update `dragPreview.type === 'vertex'` with full new points array (not just the moved vertex)
- On commit: dispatch `moveVertex(markupId, page, vertexIndex, newPoint)` per moved vertex — compare with `vertexEditOriginalRef` to detect actual changes (Pitfall 7 no-op guard)
- Click-outside commit: `handleStageClick` — if `e.target === stageRef.current` and `vertexEditMarkupId !== null`, call `commitVertexEdit()` before clearing selection

## Tasks

### T1: Add `vertexDragRef` and wire `onHandleMouseDown` on `VertexHandleOverlay`

**Type:** edit
**File:** `src/renderer/src/components/CanvasViewport.tsx`

**What:**

**Step A — Add `vertexDragRef`** (after `bodyDraggedRef`):
```typescript
// Phase 12: vertex drag state — which markup/vertex is being dragged.
type VertexDragState = {
  markupId: string
  vertexIndex: number
  origin: StagePoint        // page-space point where drag started
  originalPoints: StagePoint[]  // snapshot for Escape-restore (same as vertexEditOriginalRef)
} | null

const vertexDragRef = useRef<VertexDragState>(null)
```

**Step B — Replace the stub `onHandleMouseDown` in the `VertexHandleOverlay` mount** (added as a stub in 12-03) with the real implementation:

```jsx
onHandleMouseDown={(vertexIndex) => {
  // Start a vertex drag. The mousedown on the Rect already set e.cancelBubble=true
  // in VertexHandleOverlay, so the Stage onMouseDown will NOT fire for this gesture.
  // We use this callback (not the Stage handler) as the vertex drag entry point.
  const markup = pageMarkups.find(m => m.id === vertexEditMarkupId)
  if (!markup || markup.type === 'count') return
  const stage = stageRef.current
  if (!stage) return
  const pointer = stage.getPointerPosition()
  if (!pointer) return
  const origin = stage.getAbsoluteTransform().copy().invert().point(pointer)
  vertexDragRef.current = {
    markupId: vertexEditMarkupId,
    vertexIndex,
    origin,
    originalPoints: [...markup.points]
  }
}}
```

**Why:** The `onHandleMouseDown` callback approach is more reliable than `e.target.name().startsWith('handle-')` in the Stage handler because `e.cancelBubble = true` in the Rect's own handler prevents the Stage event from firing at all. The callback fires first (child before Stage), sets the ref, and the Stage handler never sees the event — which is the correct behavior.

**Step C — Add vertex drag detection to `handleStageMouseDown`:**

After the body-drag branch and BEFORE the rubber-band branch, add a vertex-drag guard. Since handle mousedowns have `cancelBubble = true` the Stage handler actually won't fire for handle events — BUT add a defensive check for the case where bubble-cancel was missed:

```typescript
// Phase 12: if vertexDragRef was set by onHandleMouseDown callback (child fires first),
// the Stage handler was already suppressed via cancelBubble. This guard is a safety net.
if (vertexDragRef.current !== null && activeTool === 'select') {
  e.evt.stopPropagation()
  return  // vertex drag already started via onHandleMouseDown
}
```

### T2: Implement vertex drag mousemove/mouseup and click-outside commit in `handleStageMouseMove`, `handleStageMouseUp`, and `handleStageClick`

**Type:** edit
**File:** `src/renderer/src/components/CanvasViewport.tsx`

**What:**

**Step A — `handleStageMouseMove` — add vertex drag preview BEFORE the body-drag check:**

```typescript
// Phase 12: vertex drag live preview.
const vd = vertexDragRef.current
if (vd) {
  const pt = stage.getAbsoluteTransform().copy().invert().point(pointer)
  // Build the full new points array with only the dragged vertex moved.
  const markup = pageMarkups.find(m => m.id === vd.markupId)
  if (markup && markup.type !== 'count') {
    const newPoints = markup.points.map((p, i) =>
      i === vd.vertexIndex ? pt : p
    )
    setDragPreview({ type: 'vertex', markupId: vd.markupId, vertexIndex: vd.vertexIndex, points: newPoints })
  }
  return
}
```

**Step B — `handleStageMouseUp` — add vertex drag commit BEFORE the body-drag check:**

```typescript
// Phase 12: vertex drag commit.
const vd = vertexDragRef.current
if (vd) {
  const stage = stageRef.current
  const pointer = stage?.getPointerPosition()
  if (stage && pointer) {
    const pt = stage.getAbsoluteTransform().copy().invert().point(pointer)
    const dx = pt.x - vd.origin.x
    const dy = pt.y - vd.origin.y
    const moved = Math.abs(dx) > 4 || Math.abs(dy) > 4  // D-09 4px threshold
    if (moved) {
      // The new point is the current preview position. If dragPreview has the vertex,
      // use that; otherwise compute from origin + delta.
      const newPoint = dragPreviewRef.current?.type === 'vertex'
        ? dragPreviewRef.current.points[vd.vertexIndex]
        : { x: vd.originalPoints[vd.vertexIndex].x + dx, y: vd.originalPoints[vd.vertexIndex].y + dy }
      // No-op guard: only dispatch if the point actually moved (Pitfall 7).
      const orig = vd.originalPoints[vd.vertexIndex]
      if (newPoint.x !== orig.x || newPoint.y !== orig.y) {
        useMarkupStore.getState().moveVertex(vd.markupId, currentPage, vd.vertexIndex, newPoint)
      }
    }
    // Vertex edit mode stays active after single vertex drag — user may drag another handle.
    // Only clear dragPreview (the store has the new committed positions now).
    setDragPreview(null)
    // Update the original snapshot so subsequent Escape restores from the committed state.
    const markup = useMarkupStore.getState().pageMarkups[currentPage]?.find(m => m.id === vd.markupId)
    if (markup && markup.type !== 'count') {
      vertexEditOriginalRef.current = [...markup.points]
    }
  }
  vertexDragRef.current = null
  return
}
```

  const cleanup = () => {
    if (rubberBandRef.current) setRubberBand(null)
    markupMouseDownPosRef.current = null
