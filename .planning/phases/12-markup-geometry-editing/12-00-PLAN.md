# Plan 12-00: Wave 0 — RED Test Stubs

## Goal

Create three failing test files that define the contracts for Phase 12's new commands and vertex-edit state machine; they become GREEN after Wave 1 implementation.

## Context

Phase 12 adds vertex edit mode and drag-to-translate. Before any implementation, the RED stubs lock the behavioral contracts so Wave 1 has clear pass/fail gates. This follows the established TDD pattern used in Phases 3, 4, 5, 6, 7, 8, 9, and 10.

The three test files cover:
- `move-vertex-command.test.ts` — `moveVertex` store action and undo/redo round-trips
- `move-markups-command.test.ts` — `moveMarkups` store action, solo translate, group move, count pin translate, undo/redo
- `vertex-edit-mode.test.ts` — `viewerStore` `vertexEditMarkupId` state transitions (set, clear, page-change clear)

All tests import from modules that do NOT yet export the new symbols. Every test must **fail** when run before Wave 1. Verify RED status before committing.

## Tasks

### T1: Create `src/tests/move-vertex-command.test.ts`

**Type:** create
**File:** `src/tests/move-vertex-command.test.ts`

**What:**

Write a Vitest test file using the fixture pattern from `src/tests/markup-store.test.ts`. Include a `beforeEach` that resets `useMarkupStore` to empty state (same pattern: `useMarkupStore.setState({ pageMarkups: {}, categories: {}, categoryOrder: [], undoStack: [], redoStack: [] })`). Define a `makeLinear` fixture helper that creates a `LinearMarkup` with `points: [{ x: 0, y: 0 }, { x: 100, y: 0 }, { x: 100, y: 100 }]`.

Write the following test cases inside `describe('markupStore — moveVertex command')`:

1. `'moveVertex moves the vertex at the given index and pushes move-vertex command'` — call `useMarkupStore.getState().placeMarkup(linear)` then `useMarkupStore.getState().moveVertex(linear.id, linear.page, 0, { x: 50, y: 50 })`. Assert `pageMarkups[1][0].points[0]` equals `{ x: 50, y: 50 }`. Assert `undoStack[undoStack.length - 1].type === 'move-vertex'`.

2. `'moveVertex undo restores the original vertex position'` — place markup, call `moveVertex`, call `undo()`. Assert `pageMarkups[1][0].points[0]` equals `{ x: 0, y: 0 }`. Assert `redoStack.length === 1`.

3. `'moveVertex redo re-applies the vertex move'` — place markup, `moveVertex`, `undo()`, `redo()`. Assert `pageMarkups[1][0].points[0]` equals `{ x: 50, y: 50 }`.

4. `'moveVertex is a no-op for unknown markupId'` — call `moveVertex('nonexistent', 1, 0, { x: 10, y: 10 })`. Assert `undoStack.length === 0`.

5. `'moveVertex clears redoStack'` — place markup, call undo (from place command), call `moveVertex`. Assert `redoStack.length === 0`.

The `moveVertex` action signature is: `moveVertex(markupId: string, page: number, vertexIndex: number, newPoint: StagePoint): void`.

Import `useMarkupStore` from `'@renderer/stores/markupStore'`. Import `LinearMarkup` from `'@renderer/types/markup'`. Import `StagePoint` from `'@renderer/hooks/useCalibrationMode'`.

**Why:** Locks the `moveVertex` contract before implementation. Tests must fail (RED) at this point because `moveVertex` is not yet exported from `markupStore`.

### T2: Create `src/tests/move-markups-command.test.ts`

**Type:** create
**File:** `src/tests/move-markups-command.test.ts`

**What:**

Write a Vitest test file with the same `beforeEach` reset pattern. Define fixtures:
- `makeLinear(page, id?)` — `LinearMarkup` with `points: [{ x: 0, y: 0 }, { x: 100, y: 0 }]`
- `makeCount(page, id?)` — `CountMarkup` with `point: { x: 50, y: 50 }`, `sequence: 1`

Write the following test cases inside `describe('markupStore — moveMarkups command')`:

1. `'moveMarkups translates a single linear markup'` — place linear, call `useMarkupStore.getState().moveMarkups([{ markupId: linear.id, page: 1, newPoints: [{ x: 10, y: 10 }, { x: 110, y: 10 }], oldPoints: linear.points }])`. Assert `pageMarkups[1][0].points` equals `[{ x: 10, y: 10 }, { x: 110, y: 10 }]`. Assert `undoStack[undoStack.length - 1].type === 'move-markups'`.

2. `'moveMarkups translates a count pin (point, not points)'` — place count, call `moveMarkups([{ markupId: count.id, page: 1, newPoints: [{ x: 75, y: 75 }], oldPoints: [count.point] }])`. Assert `pageMarkups[1][0].point` equals `{ x: 75, y: 75 }`.

3. `'moveMarkups batch (group move) translates multiple markups as one command'` — place two linears, call `moveMarkups` with both in the `moves` array. Assert both are moved. Assert `undoStack.length === 3` (2 places + 1 move-markups).

4. `'moveMarkups undo restores all markups in the batch'` — place two linears, group-move, undo. Assert both are back at original positions. Assert `redoStack.length === 1`.

5. `'moveMarkups redo re-applies the batch move'` — place two, group-move, undo, redo. Assert both at moved positions.

6. `'moveMarkups is a no-op for empty moves array'` — call `moveMarkups([])`. Assert `undoStack.length === 0`.

The `moveMarkups` action signature is: `moveMarkups(moves: Array<{ markupId: string; page: number; oldPoints: StagePoint[]; newPoints: StagePoint[] }>): void`.

Note: count pins use `markup.point` (singular) but the command normalises to `oldPoints: [markup.point]` / `newPoints: [newPoint]`. The undo branch reads `moves[i].oldPoints[0]` and restores to `markup.point`.

**Why:** Locks the `moveMarkups` contract for both solo translate and group move in a single command type before implementation.

### T3: Create `src/tests/vertex-edit-mode.test.ts`

**Type:** create
**File:** `src/tests/vertex-edit-mode.test.ts`

**What:**

Write a Vitest test file that tests `viewerStore` vertex-edit state. Use the same `beforeEach` reset pattern from `src/tests/viewer-store-selection.test.ts`: reset `useViewerStore` with `setState({ ..., vertexEditMarkupId: null })`.

Write the following test cases inside `describe('viewerStore — vertexEditMarkupId')`:

1. `'initialises with vertexEditMarkupId equal to null'` — assert `useViewerStore.getState().vertexEditMarkupId === null`.

2. `'setVertexEditMarkupId(id) sets vertexEditMarkupId'` — call `setVertexEditMarkupId('markup-abc')`, assert equals `'markup-abc'`.

3. `'clearVertexEdit() resets vertexEditMarkupId to null'` — set then clear, assert null.

4. `'setPage() clears vertexEditMarkupId (vertex edit is page-scoped)'` — set up with `useViewerStore.setState({ ..., totalPages: 5, vertexEditMarkupId: 'some-id' })` then call `setPage(2)`, assert `vertexEditMarkupId === null`. Without `totalPages: 5`, `setPage(2)` is a no-op and the test never goes RED.

5. `'nextPage() clears vertexEditMarkupId'` — set `useViewerStore.setState({ ..., totalPages: 5, vertexEditMarkupId: 'some-id' })`, call `nextPage()`, assert null.

6. `'prevPage() clears vertexEditMarkupId'` — set `useViewerStore.setState({ ..., totalPages: 5, currentPage: 3, vertexEditMarkupId: 'some-id' })`, call `prevPage()`, assert null.

7. `'resetViewer() clears vertexEditMarkupId'` — set then `resetViewer()`, assert null.

The `beforeEach` must include `vertexEditMarkupId: null` in the `setState` call. The `setVertexEditMarkupId` and `clearVertexEdit` actions do not yet exist — these tests will fail (RED) until Wave 1.

**Why:** Locks the `vertexEditMarkupId` store API and page-navigation lifecycle contracts before implementation.
