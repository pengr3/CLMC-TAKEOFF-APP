<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Manual UAT — 10 scenarios covering all 4 features</name>
  <what-built>
    Phase 8 delivers four user-facing changes integrated into the running app:
    1. Wall measurement tool (BrickWall icon in toolbar; draws polylines with parallel-offset hairline; m² label; popup wall-height input)
    2. Chain markup mode (after committing any markup, the tool stays armed with the same name/category/color; chip badge appears in the active toolbar button)
    3. Per-item show/hide visibility toggle (Lightbulb / LightbulbOff icon in TotalsRow; click hides markups from canvas while totals stay intact)
    4. Rifle-scope crosshair cursor (replaces the OS cursor whenever a markup or scale tool is active)
  </what-built>
  <how-to-verify>
    Run the dev build: `npm run dev` (Electron app launches). Open a sample multi-page construction PDF. Calibrate at least one page. Then walk through ALL 10 scenarios below — each is independent; a single failure blocks phase closure. Report each as PASS or FAIL with notes.

    1. **Place a wall — happy path.** Click the Wall (BrickWall) button. Cursor over canvas becomes a rifle-scope crosshair (white cross with thin black outline, ~2px gap at center). Click two points on the page; dblclick to finish. Popup appears with a "Wall height (mm)" row defaulting to 2400. Enter a name (e.g., "Internal partition"), category, color. Confirm. The wall appears as a 2.5× thick line with a thin parallel offset hairline, and the label shows m² (e.g., "12.00 m²"). Verify the m² value matches expected = (polyline length in m) × 2.4 m.

    2. **Chain a second wall.** After step 1, the Wall button shows the chain badge chip: colored dot + "Internal partition" truncated. Cursor stays as crosshair. Click two points + dblclick to draw another wall. NO popup appears — the wall commits silently with the SAME name, category, color, and height (2400 mm). Verify the second wall is labeled with its own m² value computed from its own length.

    3. **Break the chain (Esc).** Press Esc. The chain badge chip in the Wall button DISAPPEARS. Cursor stays as crosshair (Wall tool still active). Click two points + dblclick — the popup APPEARS this time (re-prompting for name + height because chain is broken).

    4. **Break the chain by re-clicking the active tool.** With chain armed, click the Wall button itself. Cursor reverts to default arrow over canvas (tool deactivated). Click Wall again to re-activate — cursor becomes crosshair, but no chain badge (fresh state).

    5. **Chain persists across PDF pages.** Place a wall on page 1 (chain arms). Use the page-next arrow to switch to page 2 mid-chain. Chain badge MUST still be visible in the Wall button. Place a wall on page 2 — no popup, commits with the same name/height as the page-1 wall.

    6. **Show/hide visibility — TotalsRow lightbulb toggle.** In the TotalsPanel, locate a row for an item with markups on the canvas. Note the Lightbulb icon to the left of the color chip. Click the lightbulb. The icon changes to LightbulbOff; ALL markups with that name disappear from the canvas. The TotalsRow's quantity, color chip, label, uom all stay unchanged. Click the lightbulb again — icon reverts; markups reappear.

    7. **Hidden state persists across save/reload.** With one or more items hidden (LightbulbOff state), Save the project (Ctrl+S). Close and re-open the .clmc file. The same items are still hidden; the LightbulbOff state is preserved.

    8. **BOQ export includes hidden items.** With one item hidden on the canvas, export to .xlsx (Ctrl+Shift+E → save dialog). Open the .xlsx in Excel. The hidden item appears in the BOQ sheet with its full quantity (D-15 — visibility is canvas-only, not an export filter).

    9. **Crosshair cursor during Set Scale.** Click "Set Scale" toolbar button. Cursor over canvas becomes the same rifle-scope crosshair (D-19 — crosshair shows for scale + verify-scale + count + linear + area + perimeter + wall). Cancel Set Scale. Cursor reverts to default.

    10. **Wall edit + undo/redo.** Right-click on a placed wall. Context menu shows Edit / Recolor Group / Delete. Click Edit. MarkupNamePopup appears in mode='edit' with the wall-height row pre-filled. Change the height (e.g., 2400 → 3000). Save Changes. The wall's label updates to show the new m² (length × 3.0). Press Ctrl+Z. Height reverts to 2400; label updates. Ctrl+Y. Height returns to 3000.

    Report for each scenario:
    - PASS / FAIL
    - If FAIL: describe what you observed and what was expected
    - If a scenario has a minor visual quirk that does NOT block the feature, note it as "PASS with note"

    If 1-2 minor issues surface (e.g., chain badge slightly cramped on narrow window, wall hairline offset visually subtle on dense plans), document in the SUMMARY.md and proceed. The fallback for wall affordance (per 08-04 documentation) is "2× stroke alone" — if the parallel hairline causes legibility issues, the fallback can ship in this phase or be deferred to a gap-closure plan.

    If a blocking failure occurs (e.g., crash, data loss, chain doesn't work, BOQ export broken), STOP. Do NOT mark the phase complete. Create a gap-closure plan via `/gsd-plan-phase 8 --gaps` after capturing the failure in a 08-UAT.md file.
  </how-to-verify>
  <resume-signal>Type "approved" if all 10 scenarios PASS (or PASS-with-note), or describe any FAIL scenarios with reproduction steps</resume-signal>
