# Phase 6 — UI Design Contract

> Visual and interaction contract for **Live View and UI Polish** (TotalsPanel, ThumbnailStrip, CanvasHeaderBar, transient row→canvas highlights). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase ships **three additive chrome surfaces** in the existing dark-theme Electron + React 19 + Konva desktop shell, plus **two transient canvas overlays** that bridge panel rows ↔ markups on the canvas. No third-party design system is introduced — this contract extends the inline-style + `COLORS` token convention already established in `Toolbar`, `StatusBar`, `MarkupContextMenu`, `MarkupTooltip`, and `ConfirmationToast`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — Electron desktop chrome uses inline styles + `COLORS` constants per established convention) |
| Preset | not applicable |
| Component library | none (pure React 19 + react-konva 19 — Konva 10 for transient canvas overlays) |
| Icon library | `lucide-react` (already installed; pinned via existing toolbar imports) |
| Font | system stack — `Inter, sans-serif` for HTML chrome (matches `MarkupTooltip`); Konva text uses default (Konva default font) |
| Styling approach | inline `style={...}` backed by `COLORS` + `LAYOUT` constants from `src/renderer/src/lib/constants.ts` — **no Tailwind in chrome path** |
| Theme | dark only (no light-mode toggle in v1) |

**Why no shadcn:** The app's chrome is already a small, internally-consistent inline-style system. Introducing shadcn would force a cascade of Tailwind setup, theme tokens, and rewrites of 20+ existing components purely to add three sidebars. The cost outweighs the benefit for a single-developer Windows desktop tool. Phase 6 extends the existing convention.

## Typography

The Phase 6 chrome reuses StatusBar / Toolbar / MarkupTooltip type ramp verbatim. **No new font sizes are introduced** beyond what is already on screen.

| Role | Size | Weight | Line Height | Phase 6 Usage |
|------|------|--------|-------------|---------------|
| Body | 13px | 400 | 1.4 | TotalsPanel item rows; thumbnail page label; CanvasHeaderBar text; tooltip body — matches `StatusBar.tsx`, `MarkupTooltip.tsx`, `Toolbar.tsx` |
| Label | 12px | 600 | 1.4 | Section labels inside `MarkupContextMenu` ("Recolor group"); reused for the Phase 6 right-click menu D-14 ("Copy as text" item label is **13/400**, the section header above it is **12/600**) — **inherited from existing `MarkupContextMenu` — not a new Phase 6 size** |
| Heading | 13px | 600 | 1.4 | TotalsPanel category-heading row text; metadata header field labels ("Project:", "Plan:") — same size/weight as Toolbar `IconButton` text |
| Display | 16px | 600 | 1.3 | Grand-total bar quantity number — one display step above body to anchor the eye; size alone (16px vs 13px) carries the visual weight, no extra font-weight tier needed |

**Numeric column rendering:** quantities right-aligned, monospace via `font-variant-numeric: tabular-nums` (no font-family change — system numerics with tabular variant). Counts shown as integer; lengths/areas shown at 2dp — mirrors BOQ export D-22 / Phase 5 D-03.

**Total declared sizes:** 4 (12, 13, 16) — within the 3-4-size budget. `12px` is a label-only size **inherited from the existing `MarkupContextMenu` pattern** and reused for the Phase 6 context-menu section header — not a net-new Phase 6 size. `13px` is the body workhorse. **Total declared weights:** 2 — 400 (body) and 600 (emphasis & display). The grand-total Display row's larger 16px size already creates strong visual separation; no third weight tier is needed.

## Layout Contract

The full app shell becomes a **three-column flex row** wrapped between the existing TitleBar, Toolbar, and StatusBar. The center column contains the new CanvasHeaderBar above the existing CanvasViewport. Both side columns are independently collapsible to a slim rail.

```
┌──────────────────────────────────────────────────────────────────┐
│                          TitleBar (32px)                         │
├──────────────────────────────────────────────────────────────────┤
│                          Toolbar (40px)                          │
├──────────┬─────────────────────────────────────────┬─────────────┤
│          │           CanvasHeaderBar (28px)        │             │
│          ├─────────────────────────────────────────┤             │
│ Thumb    │                                         │   Totals    │
│ Strip    │           CanvasViewport                │   Panel     │
│ (140px   │           (flex: 1)                     │   (320px    │
│  default)│                                         │   default)  │
│  ◐       │                                         │     ◐       │
│ resize ↔ │                                         │  resize ↔   │
├──────────┴─────────────────────────────────────────┴─────────────┤
│                          StatusBar (28px)                        │
└──────────────────────────────────────────────────────────────────┘
```

**Vertical order (no change to existing four bars):** TitleBar → Toolbar → (Strip | (Header + Canvas) | Totals) → StatusBar.

**Splitter behavior (D-03):**
- 4px hit area; visible 1px line in `COLORS.border`; hover fades to `COLORS.hoverSurface` over 100ms; mousedown shows `COLORS.accent` until release.
- `cursor: col-resize` on hover.
- Drag updates panel width in real time; commits to localStorage on `pointerup` (not on every frame).
- Min panel width: `slimRailWidth` (28px — auto-collapses to rail at this threshold). Max panel width: 50% of window inner width (caps so the panel can never starve the canvas).

**Collapse behavior (D-02, D-03):**
- Each panel has a chevron collapse-button on its **outer-edge** corner (top-outer for both panels; ThumbnailStrip's chevron points left to collapse / right to expand; TotalsPanel mirrored).
- Collapse animation: `transition: width 150ms ease-out` on the panel container. **Animation is on width only** — children do not slide independently.
- A collapsed panel becomes a **28px slim rail** showing the chevron only (no other affordance, no visible icons in the rail). Click the chevron to re-expand to last-known width.
- Default-on-first-launch: both panels open. Subsequent launches restore localStorage state.

**Persistence keys (`clmc.ui.*`, single namespaced object — Claude's discretion confirmed):**
```typescript
// Stored at localStorage key 'clmc.ui'
{
  thumbnails: { open: boolean; width: number };
  totals:     { open: boolean; width: number };
  collapsedCategories: string[];   // category names (or IDs — planner's call), folded inside TotalsPanel
}
```
- Parse failure → silent reset to defaults (`{ thumbnails: { open: true, width: 140 }, totals: { open: true, width: 320 }, collapsedCategories: [] }`).
- **Never** persist any of this in `.clmc` files — UI state is per-machine, not per-project.

**Keyboard shortcuts to toggle panels:** **none in v1** (locked discretion). The chevron buttons on panel edges are the only toggle path. Document this explicitly so the planner does not add shortcuts.

## Interaction Contract

### TotalsPanel — row interactions

**Hover an item row (D-11):**
- **Trigger:** mouse-enter on a `TotalsRow`. **0ms delay** (instant — locked discretion).
- **Effect on canvas:** every markup matching `(row.itemName + row.type)` on the **currently visible page only** is overlaid with a `<HoverRing>` on Konva Layer 2 (transient).
  - Visual: steady (non-animated) outer ring concentric to the markup's existing geometry — circle for count pin, polyline outline for linear, polygon outline for area/perimeter.
  - Color: uniform white (`COLORS.textPrimary` `#cccccc` is acceptable; `#ffffff` preferred for higher contrast over per-name colors) at **40% opacity** (locked discretion).
  - Stroke width: **2px / `currentZoom`** (zoom-compensated — divide by `currentZoom` so visual size is constant — Phase 03.1 discipline).
  - Outer ring sits **outside** the markup at +4px (in screen pixels at base zoom; therefore `4 / currentZoom` in stage units) to remain visually distinct from the pulse.
  - No fade animation — appears and disappears instantly on enter/leave.
- **Effect on panel:** the row gets `COLORS.hoverSurface` background.
- **Removes on:** mouse-leave row, page change, panel collapse.

**Click an item row (D-10, D-12):**
- **First click:** scan `pageMarkups` for the first page (lowest page number) containing ≥1 markup matching `(row.itemName + row.type)`. If matches exist on the current page, "first match" is page N₁ where N₁ ≥ currentPage; otherwise N₁ is the lowest match page. Call `viewerStore.setPage(N₁)`. After the page-set commits, fire `<PulseHighlight>` for every matching markup on N₁.
- **Subsequent clicks on the same row** (within `cycleIndexResetMs` window — recommend 30s, but planner may use "until row-leave or category-collapse" instead): cycle to N₂, N₃, … wrapping around. Cycle index lives in transient row/panel state (`useState`), **not persisted**.
- **Cycle-index indicator:** when `cycleIndex > 0`, render a 4px round `COLORS.accent` dot at the row's leading edge (between the color chip and the Item name) so the user sees "you've cycled past first match." Removes on row-leave or category-collapse.
- **Pulse visual** (locked discretion):
  - **Color:** the row's per-name color (`Markup.color` via `getColorForName(name)`) — color travels panel cell → canvas pulse → matching markup.
  - **Geometry:** ring/outline concentric to the markup, **6px / `currentZoom`** stroke width at peak (zoom-compensated). Per-markup geometry: circle for count pin (radius = pin radius + `8 / currentZoom`), polyline outline for linear, polygon outline for area/perimeter.
  - **Opacity curve:** `ease-out` from 0.85 → 0 over **1500ms total** (locked).
  - **Stroke-width curve:** linear from `6/zoom` → `2/zoom` over the fade so the ring also "deflates" visually (gives a shockwave feel without a separate scale animation).
  - **Single fade — no persistent selection state.** Pulse self-removes from canvas when fade completes; CanvasViewport owns the timer (parent-owned-lifecycle).
  - **Distinction from HoverRing:** pulse is per-name colored + animated + thicker; hover ring is white + static + thinner.
  - **No hit-target z-order elevation:** the pulse is a render-only overlay; it must not steal hover events from the underlying markup Group.

**Right-click an item row (D-14):**
- Context menu (`<TotalsRowContextMenu>`) appears at cursor position; modeled on `MarkupContextMenu` styling (background `COLORS.secondary`, border `COLORS.border`, `boxShadow: '0 4px 12px rgba(0,0,0,0.4)'`, `zIndex: 30`, defer-listener-registration pattern to avoid same-click dismissal).
- Single action: **`Copy as text`**. Payload: `{label}\t{quantity}\t{uom}` (TAB-separated, locked discretion).
  - `label` = the post-suffix BOQ row label (e.g. `Outlet (count)`, `Wall (perimeter)`).
  - `quantity` = display number per BOQ rules: integer for count, 2dp for length/area (`quantity.toFixed(2)`).
  - `uom` = the row's UoM string verbatim (`ea`, `m`, `m²`, etc.).
- After copy, fire a transient `ConfirmationToast` ("Copied {label}") with parent-owned 2000ms dismissal — same shape as the existing save toast.
- Dismiss: outside-click, Escape, or successful copy.

**Click a category heading (D-13):**
- Toggles collapse/expand of that category's items + subtotal block. The heading itself remains visible with a chevron (▸ collapsed, ▾ expanded — matches existing Toolbar Set-Scale chevron style).
- Persisted to `localStorage clmc.ui.collapsedCategories` (Claude's discretion-recommend — accepted).

### ThumbnailStrip — interactions

**Click a thumbnail tile:** `viewerStore.setPage(N)`. Active page outline updates to the new page (2px `COLORS.accent` border).

**Hover a thumbnail tile:** background lifts to `COLORS.hoverSurface`. Page label below the tile is unchanged (already always-visible).

**Right-click a thumbnail:** **no context menu in v1** (deferred — see CONTEXT.md "Out of scope").

**Visible/scroll behavior (D-17 virtualization):**
- Tiles outside the viewport render a **placeholder skeleton** (a `COLORS.secondary` filled rectangle at the page's aspect ratio).
- `IntersectionObserver` triggers rasterization when a tile enters viewport. Once rasterized, the canvas + markup overlay is cached in memory (a small per-tile React `useRef`).
- When a markup commit hits a page (D-19), the affected tile's rasterized overlay is invalidated and re-rendered on a **200ms debounced** quiet window.
- Thumbnail rasterization DPI: **48 dpi** baseline (within the 36-60 range CONTEXT.md authorized; planner may tune within that envelope after profiling).

**Per-thumbnail badges (D-16) — fixed positions on each tile:**
- **Active-page outline:** 2px `COLORS.accent` border around the entire tile when `pageNumber === viewerStore.currentPage`. Idle tiles have a 1px `COLORS.border` border.
- **Markup count chip:** top-right corner of the rasterized image, 4px inset; small numeric pill (~16px tall, padding 0 4px), background `COLORS.secondary`, border `COLORS.border`, text `COLORS.textPrimary` 11px / 600. Hidden when count is 0.
- **Scale-status icon:** top-left corner, 4px inset; 14×14px icon. **Calibrated** = `Ruler` (lucide-react) tinted `COLORS.textSecondary`. **Uncalibrated** = `AlertTriangle` (lucide-react) tinted `COLORS.warning`. Hover on icon shows a native `title` tooltip (per copywriting contract).
- **Page label:** text below the rasterized image, centered, body 13/400 (`COLORS.textPrimary` for active page, `COLORS.textSecondary` for non-active). Resolved via `pdfDocument.getPageLabels()` first, fallback to `Page {N}`.

### CanvasHeaderBar — interactions

- **Left segment:** current page label (`Page {N}` or resolved getPageLabels label). Body 13/400, `COLORS.textPrimary`. Padded 16px from the bar's left edge.
- **Right segment:** scale status. 16px from the bar's right edge.
  - **Calibrated:** text `1:{N}` (formatted via `formatScaleRatio`), `COLORS.textPrimary`.
  - **Uncalibrated AND no non-count markups on this page:** text `Not Set`, `COLORS.textSecondary` (silent — no nudge).
  - **Uncalibrated AND ≥1 non-count markup on this page:** text `Page not calibrated.` followed by inline link `Set Scale`. Link styling: `COLORS.accent`, underline, `cursor: pointer`. Click invokes the same path Toolbar's Set-Scale uses (`getCalibrationControls()?.activate()`) — **must NOT duplicate the calibration trigger code** (D-19 / Claude's-discretion-locked).
- Bar height: 28px (`canvasHeaderBarHeight`). Background: `COLORS.secondary`. Border-bottom: 1px `COLORS.border` (matches StatusBar `borderTop`). Border-top: 1px `COLORS.border` (sits below the Toolbar's own border-bottom; the visual stack is consistent).
- The existing bottom StatusBar is **retained unchanged** — CanvasHeaderBar is additive; experienced users still glance at the bottom for zoom %.

### Transient overlay coexistence

`<PulseHighlight>`, `<HoverRing>`, `<MarkupTooltip>`, and `<MarkupContextMenu>` all render through `CanvasViewport`. Z-order (bottom → top):
1. PDF Image (Layer 0)
2. Calibration overlay + in-progress linear preview (Layer 1a, non-listening)
3. Committed markups (Layer 1b, listening=true)
4. **`<HoverRing>`** (Layer 2 transient, non-listening)
5. **`<PulseHighlight>`** (Layer 2 transient, non-listening)
6. Right-click menu / hover tooltip (HTML, `zIndex: 25-30`)

HoverRing and PulseHighlight must `listening={false}` so neither steals hover events from the underlying markup Group (which would break the existing tooltip flow).

## Constraints (carried from earlier phases — must not regress)

1. **Markup pinning at any zoom level (CLAUDE.md, Phase 1 success criterion).** HoverRing + PulseHighlight zoom-compensated stroke widths (`/ currentZoom`) are mandatory. Any regression of pin precision at zoom is a release blocker.
2. **Live-view = export-view (D-04, D-07).** TotalsPanel renders the same `BoqStructure` produced by the aggregator that feeds the .xlsx writer. **One aggregator. Zero divergence.** A change to BOQ export must propagate to TotalsPanel (and vice versa).
3. **Color is per-name-group, applied as a chip on Item-name cell only (D-06).** Categories never carry color. Subtotal / grand-total / metadata rows never carry per-name color.
4. **localStorage namespace `clmc.ui.*`.** Never persist UI panel state in `.clmc` project files (D-03).
5. **Inline-style + COLORS tokens convention** (StatusBar, Toolbar, all popups, all modals). No Tailwind in chrome path.
6. **Parent-owned-lifecycle for transient UI** (`MarkupTooltip`, `ConfirmationToast`). Pulse + HoverRing follow the same pattern: CanvasViewport owns timers / mount / unmount; the visual components are pure presentational.
7. **Module-level stable empty-array fallback** (`EMPTY_MARKUPS` pattern, commit `0e1a8e0`). TotalsPanel selectors must use stable fallbacks; never invoke methods inside Zustand selectors.
8. **Layer split** (Layer 1a non-listening + 1b listening + Layer 2 transient). PulseHighlight and HoverRing render on Layer 2 with `listening={false}`.
9. **Thumbnail virtualization via `IntersectionObserver`** (D-17). 50+ page PDFs must not regress open-time. Thumbnail rasterization DPI 36-60 dpi range.
10. **Debounced ~200ms thumbnail-overlay refresh on markup commit (D-19).** Hooks into markupStore subscription; no thrash during continuous editing.
