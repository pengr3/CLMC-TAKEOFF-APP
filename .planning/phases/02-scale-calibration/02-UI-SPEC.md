# Phase 2 — UI Design Contract

> Visual and interaction contract for the Scale Calibration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase extends the Phase 1 design system. All tokens (color, spacing, typography, font stack, icon library) are inherited from `01-UI-SPEC.md` unchanged. This document only declares what is NEW or CHANGED for Phase 2.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | Inherited from Phase 1 |
| Preset | not applicable | Inherited from Phase 1 |
| Component library | none (hand-rolled components, inline styles) | Inherited from Phase 1 |
| Icon library | Lucide React (`lucide-react`) | Inherited from Phase 1 |
| Font | Inter 400 + 600 (`@fontsource/inter`) | Inherited from Phase 1 |

**Rationale:** Phase 2 adds one new toolbar button, one new status-bar segment, one temporary Konva `<Line>` decoration, and one inline HTML popup overlay. This does not justify introducing shadcn/Radix — all four additions follow patterns already established in Phase 1 (`IconButton`, status-bar segment, Konva overlay, inline styles). shadcn remains a candidate for Phase 3 when the first true dialog appears.

**New dependencies:** none. The popup is a React `<div>` with `position: absolute`; no floating-ui / popperjs required (RESEARCH.md §"No New Dependencies").

---

## Layout

Phase 2 reuses the Phase 1 three-region layout (title bar → toolbar → canvas → status bar) with two additions:

1. A new **Set Scale** toolbar button, positioned on the left side of the toolbar, immediately after the existing "Open PDF" button.
2. A new **Scale** segment in the status bar, positioned after the existing "Zoom" segment on the right.

The calibration popup is a transient, absolutely-positioned HTML overlay that appears inside the canvas viewport container — it is not part of the fixed layout chrome.

```
+-------------------------------------------------------+
|  Title Bar (32px)                                     |  (unchanged)
+-------------------------------------------------------+
|  Toolbar (40px)                                       |
|  [Open PDF] [Set Scale] | [<] P3/12 [>] | Zoom 125%  |  NEW: Set Scale button
+-------------------------------------------------------+
|                                                       |
|              Canvas Viewport                          |
|                                                       |
|                  (calibration line + popup live here) |
|                                                       |
+-------------------------------------------------------+
|  Status Bar (28px)                                    |
|  file.pdf | Page 3/12 | Zoom: 125% | Scale: 1:100    |  NEW: Scale segment
+-------------------------------------------------------+
```

### Visual Hierarchy

- **When current page is uncalibrated:** The "Set Scale" toolbar button is the focal action (painted in accent color to draw attention — see Color §Accent Usage). The status bar "Scale: Not Set" in amber reinforces the same signal.
- **During calibration (drawing state):** The canvas cursor changes to `crosshair` and the calibration line renders in accent color as the user moves between click 1 and click 2. All other chrome recedes.
- **During calibration (confirming state):** The inline popup near the line endpoint is the focal element. The popup's Confirm button is the primary action.
- **When current page has a scale set:** "Set Scale" returns to the standard neutral toolbar button style. Status bar shows "Scale: 1:N" in primary text color.

`#0078d4` is reserved for — and ONLY for — these specific elements:

1. "Open PDF" button background (Phase 1, unchanged)
2. Zoom percentage text when zoom is not 100% (Phase 1, unchanged)
3. Active page indicator (Phase 1, unchanged)
4. Drop-zone border when dragging a file over the empty state (Phase 1, unchanged)
5. **NEW: The calibration line stroke color during drawing and confirming states**
6. **NEW: The computed ratio preview text inside the popup when the line length and distance are valid**

### Warning Color Usage (new `#f0a500`)

`#f0a500` is reserved for — and ONLY for — these elements:

1. "Scale: Not Set" text in the status bar when the current page is uncalibrated
2. A 1px ring/glow around the "Set Scale" toolbar button when the current page is uncalibrated and a PDF is loaded (draws attention to the required action)
3. Inline popup error text when the drawn line is below the minimum 10-pixel threshold (RESEARCH.md Pitfall 6)

**Destructive:** not used in Phase 2. Re-calibration replaces scale silently (no confirmation required per CONTEXT.md locked decision: "Re-calibration: clicking Set Scale again lets user draw a new line, which replaces the existing scale"). No delete-all flow in this phase.

## Component Inventory

### NEW: Set Scale Toolbar Button

Follows the Phase 1 `IconButton` (icon + label variant) pattern exactly, with a state-dependent outline.

- Icon: `Ruler` from Lucide (`import { Ruler } from 'lucide-react'`) — 16x16px, `currentColor`
- Label: `"Set Scale"` — 13px semibold (600)
- Height: 28px, padding 4px 8px, border-radius 4px, icon-label gap 4px
- **Default state (page has a scale set):** background `transparent`, text `#cccccc`, border `none`
- **Hover:** background `#2d2d30`
- **Active/pressed:** background `#37373d`
- **Highlight state (page is uncalibrated AND a PDF is loaded):** background `transparent`, text `#cccccc`, `box-shadow: 0 0 0 1px #f0a500` (subtle 1px warning ring, no layout shift)
- **Active-mode state (Set Scale calibration mode is engaged):** background `#0078d4`, text `#ffffff` (matches Open PDF button treatment — signals "this mode is armed; next click starts drawing")
- **Disabled (no PDF loaded):** opacity 0.4, cursor not-allowed, no highlight ring
- Position in toolbar: left cluster, immediately right of the Open PDF button, with an 8px gap

### NEW: Calibration Line (Konva decoration, not persisted)

Rendered as a `<Line>` in Layer 1 of the Konva Stage during `drawing` and `confirming` states. Not persisted to state; removed on Confirm or Cancel.

- Stroke: `#0078d4` (accent)
- Stroke width: `2 / stage.scaleX()` (kept visually constant at 2px regardless of zoom — RESEARCH.md Pattern 2)
- Line cap: `round`
- Dash pattern: `[8 / stage.scaleX(), 4 / stage.scaleX()]` (dashed to distinguish from future solid markup lines)
- Endpoint markers: small filled circles at start and end points
  - Radius: `4 / stage.scaleX()`
  - Fill: `#0078d4`
  - Stroke: `#ffffff`, stroke width `1 / stage.scaleX()`
- Preview line during `drawing` state (after click 1, before click 2): the line's end point follows the cursor in real time; line remains styled identically

### NEW: Scale Popup (inline HTML overlay)

A React `<div>` with `position: absolute`, positioned inside the canvas viewport container at the projected screen coordinate of the line endpoint plus an 8–12px offset (clamped to container bounds).

**Container:**

- Background: `#252526` (secondary surface)
- Border: `1px solid #3c3c3c`
- Border-radius: 8px
- Padding: 16px (md)
- Min-width: 220px, max-width: 260px
- Box-shadow: `0 4px 12px rgba(0, 0, 0, 0.4)` (subtle elevation to separate from canvas)
- z-index: 10 (above Konva Stage)
- The popup is a focus trap while open (Tab and Shift+Tab cycle within the popup only)

**Layout (vertical stack, 8px gap between rows):**

```
+----------------------------------------+
| Enter real-world distance              |  ← helper hint, 13px regular #808080
|                                        |
| Distance                               |  ← label, 13px semibold #cccccc
| [ 5.00                ]  [ m    v ]   |  ← input + unit dropdown
|                                        |
| Scale: 1:100                           |  ← computed ratio preview, 13px semibold #0078d4
|                                        |
|                  [ Cancel ] [ Confirm ] |  ← action row, right-aligned
+----------------------------------------+
```

**Fields:**

- **Helper hint** (top row): "Enter real-world distance" — 13px regular 400, `#808080`
- **"Distance" label** + text input:
  - Label 13px semibold 600 `#cccccc`, 4px bottom margin
  - Input: `<input type="text" inputMode="decimal">`, height 28px, padding 4px 8px, background `#1a1a1a`, border `1px solid #3c3c3c`, border-radius 4px, text 13px regular `#cccccc`
  - Focus: border color changes to `#0078d4`
  - Autofocuses when popup opens
  - Placeholder: `"0.00"`
- **Unit dropdown:**
  - Native `<select>`: height 28px, padding 4px 8px, same background/border styling as input
  - Options: `mm`, `cm`, `m`, `in`, `ft`
  - Default value: bound to `scaleStore.globalUnit` (RESEARCH.md Pitfall 5)
- **Computed ratio preview:** "Scale: 1:{N}" — 13px semibold 600, color `#0078d4` when valid, `#f0a500` when line too short or distance ≤ 0, `#808080` when distance field is empty
- **Inline error** (replaces ratio preview when line is too short): "Line too short — please draw again." — 13px regular 400, `#f0a500`
- **Action row:**
  - Right-aligned, 8px gap between Cancel and Confirm
  - **Cancel button:** neutral toolbar-button styling — background transparent, text `#cccccc`, 13px semibold, padding 4px 12px, border `1px solid #3c3c3c`, border-radius 4px, hover `#2d2d30`, active `#37373d`
  - **Confirm button:** accent CTA styling — background `#0078d4`, text `#ffffff`, 13px semibold, padding 4px 12px, border-radius 4px, hover `#1a86db`, active `#0067b8`
  - Confirm is disabled (opacity 0.4, not-allowed cursor) when distance is empty, non-numeric, ≤ 0, or pixel length < 10

### NEW: Status Bar Scale Segment

A fourth segment appended to the existing status bar, right of the Zoom segment, separated by the same 1px `#3c3c3c` divider.

- Label text: `"Scale: "` — 13px regular 400, `#cccccc`
- Value text:
  - When scale is set: `"1:{N}"` (e.g. `"1:100"`) — 13px regular 400, `#cccccc`
  - When no scale: `"Not Set"` — 13px regular 400, `#f0a500`
  - When no PDF loaded: em-dash `"—"` (matches other segments)
- No icon in this phase
- Follows the existing `Divider` component pattern (16px vertical, 1px wide, 16px margin on each side)

### NEW: Verify Scale Control

SCAL-04 requires the user to measure a second known dimension to confirm calibration accuracy. Two UI affordances deliver this:

1. A small **"Verify"** text link appears inside the popup AFTER a successful Confirm, as a transient inline confirmation toast. The toast reads: `"Scale set to 1:{N}. Verify by measuring a second line?"` with a "Verify" link-button and a "Dismiss" link-button. Toast auto-dismisses after 6 seconds or on Dismiss.
2. A secondary **Verify Scale** affordance is NOT added to the toolbar in Phase 2 (to keep the toolbar compact). Instead: once a page has a scale, right-clicking the "Set Scale" toolbar button opens a mini menu with two items: "Recalibrate" (same as left-click) and "Verify scale". The context menu is styled identically to a minimal popup (`#252526` bg, `#3c3c3c` border, 4px padding, 13px regular text, hover `#2d2d30`).

**Verify mode flow:**

- User selects "Verify scale" → cursor changes to crosshair → click-click to draw a second line → popup appears at line endpoint with content: `"Measured: 4.97 m (expected: ? )"` and a single "Dismiss" button. There is no editable field — this mode only reports the measured real-world distance derived from the existing `pixelsPerUnit`. The user mentally compares this to the known dimension they just traced.

### MODIFIED: Canvas Viewport (cursor behavior)

The canvas viewport cursor in Phase 2 gains two new states:

| State | Cursor |
|-------|--------|
| No PDF loaded | `default` (Phase 1) |
| Hovering canvas with PDF, no tool active | `grab` (Phase 1) |
| Panning (space+drag or middle-drag) | `grabbing` (Phase 1) |
| **Set Scale mode engaged, not yet drawing** | `crosshair` (NEW) |
| **Set Scale mode, between click 1 and click 2** | `crosshair` (NEW) |
| **Verify mode engaged** | `crosshair` (NEW) |

The crosshair cursor is the operating-system native crosshair (`cursor: crosshair` — no custom image). Spacebar-pan still works during calibration mode (RESEARCH.md Anti-Pattern: "Blocking spacebar pan during calibration").

## Interaction Contracts

### Set Scale Flow (SCAL-01)

1. User clicks the "Set Scale" toolbar button.
2. Button flips to active-mode state (accent background, white text). Cursor changes to crosshair when hovering canvas.
3. User clicks on the plan at the start of a known dimension. A 4px-radius accent dot appears at that point. Line preview follows the cursor.
4. User clicks at the end of the known dimension. Line finalizes. If pixel length < 10 px, popup opens but shows the inline error `"Line too short — please draw again."` and Confirm is disabled; user must press Cancel and retry.
5. If pixel length ≥ 10 px, popup opens at the endpoint. Distance input is autofocused. Unit dropdown shows `scaleStore.globalUnit` (defaults to `m` on first ever calibration).
6. User types distance, optionally changes unit. The computed ratio preview updates live as the user types.
7. User clicks Confirm (or presses Enter). `scaleStore.setScale(currentPage, pixelsPerUnit, unit)` is called. `globalUnit` is updated to the chosen unit. Popup dismisses. Calibration line is removed. Toolbar button returns to normal state. Status bar updates to `"Scale: 1:{N}"`.
8. Confirmation toast appears (see Verify Scale Control §1) with option to verify.

### Cancel Calibration Flow

At any point during `drawing` or `confirming`:

- User presses **Escape** → calibration line cleared, popup dismissed, mode returns to `idle`, toolbar button returns to neutral state.
- User clicks the **Cancel** button in the popup → same as Escape.
- User clicks the **"Set Scale" toolbar button a second time while in drawing mode** → calibration line cleared, mode returns to `idle` (button toggles off).

### Verify Scale Flow (SCAL-04)

1. Either: (a) user clicks "Verify" in the post-confirmation toast, OR (b) user right-clicks "Set Scale" toolbar button and selects "Verify scale".
2. Cursor changes to crosshair. User click-clicks a second line over a known dimension.
3. Popup appears at line endpoint showing `"Measured: {realWorldValue} {unit}"` — derived from `pixelsToRealWorld(pixelLength, pageScale.pixelsPerUnit)`, rounded to 2 decimal places.
4. User dismisses popup. No state is mutated during verify — this flow is read-only.

### Re-calibration Flow

If the current page already has a scale and the user clicks "Set Scale":

- Standard Set Scale Flow runs exactly as §Set Scale Flow above.
- On Confirm, the previous `pixelsPerUnit` is overwritten without warning (CONTEXT.md locked decision).
- The status bar updates to the new ratio immediately.

### Scale Display — Page Change

When the user navigates to a different page, the status bar Scale segment updates to reflect that page's scale (or "Not Set" in amber if that page is uncalibrated). This is a `useScaleStore` subscription via `getScale(currentPage)` — no additional interaction.

### Keyboard Shortcuts (Phase 2 additions)

| Shortcut | Action |
|----------|--------|
| Escape (during calibration) | Cancel calibration, dismiss popup, exit Set Scale mode |
| Enter (popup focused with valid distance) | Confirm |
| Tab / Shift+Tab (popup focused) | Cycle focus within popup (focus trap) |

**Intentional omission:** No global keyboard shortcut to activate Set Scale mode in Phase 2. The toolbar click is the only activation. A dedicated shortcut (e.g. `S`) may be added in Phase 6 alongside other productivity shortcuts when the shortcut map stabilises. This aligns with Phase 1's pattern of explicit tool activation.

## Accessibility Notes

### Toolbar Set Scale button

- `aria-label="Set Scale — draw a line over a known dimension"` (matches tooltip)
- `aria-pressed="true"` when calibration mode is active, `aria-pressed="false"` otherwise
- `aria-disabled="true"` when no PDF is loaded
- Keyboard focus ring: 2px solid `#0078d4`, 2px offset (Phase 1 baseline)

### Scale popup

- `role="dialog"`, `aria-modal="true"` (focus trap — see Interaction Contracts)
- `aria-labelledby` points at a visually-hidden heading `"Set scale for page {N}"`
- Distance input: `<label htmlFor="scale-distance">Distance</label>` + `aria-describedby` pointing at the computed ratio line so screen readers announce the ratio as the user types
- Inline error messages use `role="alert"` so assistive tech announces them on appearance
- Escape key closes popup in addition to Cancel button
- Enter key submits when focus is on distance input AND distance is valid

### Status bar Scale segment

- Wrapped in `aria-live="polite"` region so scale changes are announced without stealing focus
- When "Not Set", the full announced string is `"Scale: Not set"` (readable phrasing)

### Calibration line visibility

- Dashed accent line at 2px visual width is visible on all documented PDF backgrounds (white paper, scanned grey, CAD black)
- Not the sole channel of information: the popup conveys the full state in text

## HiDPI / Display Scaling

No changes from Phase 1. All new Phase 2 dimensions (popup sizes, input heights, button padding) are in CSS pixels — Electron/Chromium handles DPI scaling for HTML chrome. The calibration line's stroke-width divided by `stage.scaleX()` is unrelated to DPI and is already correct (RESEARCH.md Pattern 2).

**Verified concern:** At 150% Windows display scaling, the 220px popup min-width renders at 330 physical pixels, which is comfortable for the two-field + button-row layout. No special handling required.
