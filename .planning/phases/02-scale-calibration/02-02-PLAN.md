    **1. Create `src/renderer/src/hooks/useCalibration.ts`:**

    A custom hook managing calibration interaction state. Accepts `stageRef: RefObject<Konva.Stage | null>`.

    State:
    - `calibrationPoints: CalibrationPoint[]` (0, 1, or 2 points) — component-level useState, reset on page change or tool change
    - `showDialog: boolean` — true when 2 points placed and pixelDistance computed
    - `pixelDistance: number | null` — computed after 2nd point placed
    - `verifyResult: { pixelLength: number; realWorldLength: number } | null` — for verify-scale mode

    Read from store: `activeTool`, `currentPage`, `getPageScale`, `setPageScale`, `setActiveTool`

    Core logic:
    - `handleStageClick(e: Konva.KonvaEventObject<MouseEvent>)`:
      - If `activeTool` is not `'scale'` or `'verify-scale'`, return early
      - If `e.evt.button !== 0`, return (only left-click)
      - Get page-space coordinates using `stage.getAbsoluteTransform().copy().invert().point(stage.getPointerPosition()!)` — this inverts the Stage zoom/pan transform to get PDF-page-space coords
      - If `activeTool === 'scale'`:
        - First click: push point to calibrationPoints
        - Second click: push point, compute `pixelDistance` via `euclideanDistance(p1.x, p1.y, p2.x, p2.y)`, set `showDialog = true`
      - If `activeTool === 'verify-scale'`:
        - Same two-click pattern, but after second click: get `pageScale = getPageScale(currentPage)`, if null show alert "Calibrate first", else compute `realWorldLength = pixelsToRealWorld(pixelDistance, pageScale.pixelsPerUnit)`, set `verifyResult`

    - `handleDialogConfirm(distance: number, unit: MeasurementUnit)`:
      - `const ppu = computePixelsPerUnit(pixelDistance!, distance)`
      - Build `ScaleState`: `{ pixelsPerUnit: ppu, unit, realWorldDistance: distance, linePoints: [p1.x, p1.y, p2.x, p2.y] }`
      - Call `setPageScale(currentPage, scaleState)`
      - Reset: `setActiveTool('select')`, clear calibrationPoints, showDialog=false

    - `handleDialogCancel()`: Reset calibrationPoints, showDialog=false, keep activeTool as-is so user can retry

    - `dismissVerifyResult()`: Clear verifyResult, reset calibrationPoints, setActiveTool('select')

    - Reset calibrationPoints when `currentPage` changes or `activeTool` changes (useEffect dependencies)

    - `cancelCalibration()`: Reset calibrationPoints, setActiveTool('select') — for Escape key

    Return: `{ calibrationPoints, showDialog, pixelDistance, verifyResult, handleStageClick, handleDialogConfirm, handleDialogCancel, dismissVerifyResult, cancelCalibration }`

    - Styling: dark modal matching app theme (bg COLORS.secondary, border COLORS.border, text COLORS.textPrimary, accent button COLORS.accent). Centered overlay with semi-transparent backdrop.

    Props: `{ pixelDistance: number; onConfirm: (distance: number, unit: MeasurementUnit) => void; onCancel: () => void }`

      - When 2 points exist: `<Line points={[p1.x, p1.y, p2.x, p2.y]} stroke="#ff4444" strokeWidth={2 / currentZoom} dash={[8 / currentZoom, 4 / currentZoom]} />`
      - For existing calibration line (when page has scale and activeTool is 'select'): render a faint reference line using `getPageScale(currentPage)?.linePoints` with `stroke="#ff4444" opacity={0.3} strokeWidth={1.5 / currentZoom} dash={[6 / currentZoom, 3 / currentZoom]}`
    - Get `currentZoom` from `getViewport(currentPage).zoom`
