<decisions_made>

- **pixelsPerMm as unit-agnostic base**: PageScale now stores `pixelsPerMm: number` + `displayUnit: ScaleUnit` instead of `pixelsPerUnit: number` + `unit: ScaleUnit`. This means changing globalUnit (e.g. metres → feet) only affects display/input — stored calibration values stay correct. MM_PER_UNIT conversion table in scale-math.ts handles display conversions.

- **Reactive calibMode in scaleStore**: `calibMode: CalibMode` added to ScaleState with a `setCalibMode` action. useCalibrationMode hook calls setCalibMode on every transition. Toolbar subscribes via `useScaleStore((s) => s.calibMode)`. The 100ms setInterval polling was eliminated. The module-level ref (_calibrationControls) is retained only for imperative activate/cancel calls.

- **Persistent ConfirmationToast**: No auto-dismiss timer (no setTimeout, no AUTO_DISMISS_MS). CanvasViewport dismisses the toast when (a) user clicks Verify/Dismiss, (b) currentPage changes, or (c) a new calibration starts (calibState.mode === 'drawing').

- **LOW concerns handled**: formatScaleRatio uses `Number(ratio.toFixed(1))` for one decimal place when non-integer (avoids cumulative rounding errors). Popup edge clipping (OS window) deferred to v2. Right-click discoverability addressed with a small ▾ chevron on the Set Scale button when page is calibrated.
</decisions_made>
