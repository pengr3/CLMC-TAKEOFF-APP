# Phase 03.1 Plan 05: Hover Tooltip + Context-Menu Recolor Summary

**Committed markups are now discoverable (D-30 hover tooltip shows the name + measurement after a 200ms dwell) and post-placement recolor ships via a right-click context menu that re-uses the ScaleContextMenu dismissal pattern and writes through markupStore.recolorGroup so the whole name-group flips in a single undoable command (D-28/D-29).**

## Accomplishments

- **D-30 hover tooltip** — HTML-absolute-positioned tooltip appears after a 200ms dwell over any committed markup, disappears immediately on mouseleave, and carries the markup's name plus a measurement summary when the page is calibrated. Built as a pure presentational component so the parent owns timing (matches the ConfirmationToast pattern).
- **D-28 + D-29 right-click context menu** — 10-swatch recolor row + Delete item on every committed markup. Swatch click calls `useMarkupStore.getState().recolorGroup(markup.name, hex)` so every existing markup with the same name retro-applies the color in a single undoable `recolor-group` command. Delete calls `deleteMarkup(page, id)` for per-markup removal.
- **D-33 HTML absolutely-positioned tooltip** — escapes canvas bounds cleanly, uses the existing popup/menu styling vocabulary (dark dominant background, border, drop shadow, 13px body font). `pointerEvents: 'none'` prevents it from stealing hover from the underlying Konva Group.
- **Listening layer split in CanvasViewport** — Layer 1 was `listening={false}` before; now Layer 1a (calibration + in-progress linear preview) stays non-listening and Layer 1b (committed markups only) flips to `listening={true}`. Cost: zero for preview-only frames; the only new hit-test work is the committed-markup Groups, and only while they're on-page.
- **Forward-compat markup event props** — all four markup components now accept optional `onHoverEnter / onHoverLeave / onContextMenu`. Components render identically when the props are omitted, so future plans (edit mode, selection rings) can hook into the same event surface without re-plumbing.
- **ScaleContextMenu parity** — `MarkupContextMenu` re-uses the same `useRef` + `useEffect` outside-click + Escape dismissal, deferred registration via `setTimeout(0)` so the triggering right-click doesn't immediately close the menu.
- **6 new vitest tests** (201/201 total, up from 195) locking: 10-swatch count + Delete menuitem, current-color aria-checked exclusivity, swatch click -> `onRecolor(hex)` + `onClose`, Delete click -> `onDelete` + `onClose`, Escape -> `onClose` only, and `recolorGroup` integration (name-scoped: other name-groups stay untouched).

## Decisions Made

See "key-decisions" in frontmatter. Key highlights:

- **Layer 1 split (1a non-listening + 1b listening)** — keeps the non-interactive calibration/preview path entirely off the hit-testing graph.
- **Hover debounce lives at the parent (CanvasViewport)** — `MarkupTooltip` is pure presentational per the plan's interface spec.
- **Tooltip summary built inline via `buildMarkupSummary`** — smaller blast radius than exposing measurement helpers from each markup component; CanvasViewport already owns pageScale and per-markup lookups.
- **Context menu `currentColor` wired to `contextMarkup.color`** (not `getColorForName`) — the pin the user right-clicked is authoritative.
- **Test file kept as `.test.ts`** with `React.createElement` to stay inside the existing vitest include glob (`src/tests/**/*.test.ts`), matching `markup-namepopup.test.ts` precedent.
