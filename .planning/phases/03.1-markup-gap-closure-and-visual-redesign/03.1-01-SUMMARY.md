## Decisions Made

- **oldColors as dict rather than single oldColor** — preserves per-pin color drift on undo. A user who ad-hoc recolored pin 3 to green before doing a name-group recolor-to-blue then undoing the recolor still gets pin 3 back to green (not the pre-ad-hoc red). Matches the "undo does exactly one step" invariant.
- **getColorForName most-recent wins** — if a user has 5 old red pins and their most-recent pin was blue, `getColorForName` returns blue. Rationale: matches how users reason about "what color am I placing right now" without needing to search the history.
- **Tailwind 600-level palette** — already WCAG-AA against white AND black, so D-23 auto-contrast has no edge cases. Ten swatches chosen (top of D-26's 8-12 range) for maximum name-group capacity before palette exhaustion on a dense page.
- **Category.color retained** — removing it would cascade breakage across the 4 markup rendering components currently reading `category.color`. Those components will migrate to `markup.color` in plan 03.1-02. Keeping Category.color alive during migration prevents a 300-line cross-file edit in this plan.
