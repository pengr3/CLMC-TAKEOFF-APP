- **B2 fix landed:** LinearMarkup label now sits at `polylineMidpointByArcLength(points)` - the true geometric midpoint walked by cumulative segment distance. No more arbitrary vertex picks on multi-segment lines.

- **Measurement-only labels (D-31):** Linear shows "12.4 m"; Area shows "38.2 m²"; Perimeter shows "P: 24.6 m  A: 38.2 m²". Two-line labels replaced with single-line measurement-focused text.

- **Groups opened up for plan 05:** Outer `<Group>` on all four components no longer has `listening={false}`, so plan 03.1-05 can attach `onMouseEnter`/`onMouseLeave`/`onContextMenu` without touching these files again.

## Decisions Made

- **0.179 luminance threshold as a module-level literal** - The acceptance criteria required the exact string `CONTRAST_LUMINANCE_THRESHOLD = 0.179` for grep-based verification. Implemented as a module-level const so the value is greppable and the intent (WCAG 4.5:1 contrast break point) lives in the adjacent comment.
- **Module-level pin geometry constants instead of inline** - PIN_RADIUS_WORLD / PIN_STROKE_WORLD / NUMBER_FONT_WORLD hoisted out of the component body. Makes a future "accidentally re-added `/ currentZoom`" regression visible in a single-line diff and prevents the constants from being treated as stale JSX literals.
- **Conditional label rendering `{labelText && <Text>}`** - Linear/Area/Perimeter no longer render an empty `<Text>` node when `pageScale` is null. Keeps the Konva scene graph minimal on uncalibrated pages.
- **Kept `category: Category` in all four component props** - Legacy prop compat. CanvasViewport still passes `category={category}` to each. Removing the prop would cascade through CanvasViewport lines 477-533 and the `getCategory(m.categoryId)` lookup path; scope of this plan was the render logic only. Category retains its role as the grouping label for BOQ export (D-27) even though color is now markup-owned.
- **Inner `<Text>` inside CountPinMarkup Group keeps `listening={false}`** - The pin number shouldn't separately intercept hover events; the Circle carries the hit area. Outer `<Group>` drops `listening={false}` so plan 05 hover handlers on the Group fire when the mouse is anywhere over the pin (circle OR number).
- **Stripped the descriptive comment that mentioned `category.color`** - Initial draft had "`Colored circle using markup.color (D-29 - not category.color)`" in the JSDoc. The acceptance criteria grep for "does NOT contain the substring `category.color`" would have failed on the explanatory comment. Rewrote as "`Colored circle using markup.color per D-29`" to satisfy the gate without losing the intent.
