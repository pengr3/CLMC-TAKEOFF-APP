# Phase 1 — UI Design Contract

> Visual and interaction contract for the PDF Viewer and Canvas Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Tailwind utility classes only) |
| Icon library | Lucide React (lucide-react) |
| Font | Inter (bundled, loaded locally for offline use) |

**Rationale:** Phase 1 has fewer than 10 UI elements outside the canvas. Tailwind CSS 4.x handles the dark toolbar and status bar without a component library. shadcn/Radix can be introduced in Phase 3 or 6 when dialogs, dropdowns, and panels appear. Lucide is the standard icon set used by shadcn and provides the exact icons needed (FileUp, ChevronLeft, ChevronRight, ZoomIn, ZoomOut, Maximize). Inter is a professional sans-serif that renders crisply at small sizes on Windows.

---

## Layout

Phase 1 uses a three-region layout. All dimensions are fixed, not responsive -- this is a desktop app at a known minimum resolution of 1920x1080 at 100% scale (or 1280x720 at 150% scale).

```
+-------------------------------------------------------+
|  Title Bar (Electron frameless, 32px)                  |
+-------------------------------------------------------+
|  Toolbar (40px height)                                 |
|  [Open PDF]  |  [< Prev] Page 3/12 [Next >]  | Zoom  |
+-------------------------------------------------------+
|                                                        |
|                                                        |
|              Canvas Viewport                           |
|              (remaining height)                        |
|              PDF rendered as Konva Image               |
|              on bottom layer                           |
|                                                        |
|                                                        |
+-------------------------------------------------------+
|  Status Bar (28px height)                              |
|  filename.pdf  |  Page 3 of 12  |  Zoom: 125%         |
+-------------------------------------------------------+
```

| Region | Height | Background | Purpose |
|--------|--------|------------|---------|
| Title bar | 32px | `#1e1e1e` | Electron custom title bar with window controls (minimize, maximize, close) |
| Toolbar | 40px | `#252526` | Open PDF button, page navigation, zoom controls |
| Canvas viewport | `calc(100vh - 100px)` | `#1a1a1a` (visible when no PDF loaded or around page edges) | PDF + Konva Stage fills this region |
| Status bar | 28px | `#252526` | Current filename, page number, zoom percentage |

**Title bar:** Use Electron's `titleBarStyle: 'hidden'` with a custom-drawn title bar div. The title bar is draggable (`-webkit-app-region: drag`). Window control buttons (minimize, maximize, close) are positioned at the top-right and are NOT draggable (`-webkit-app-region: no-drag`).

### Visual Hierarchy

- **When no PDF is loaded:** The 48px FileUp icon in the centered empty state card is the focal point. The toolbar and status bar are secondary, with the "Open PDF" accent button drawing the eye toward the primary action.
- **When a PDF is loaded:** The canvas viewport is the focal point, occupying the vast majority of screen space. The toolbar and status bar recede into peripheral chrome.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap within toolbar buttons |
| sm | 8px | Toolbar button internal padding, status bar item gaps |
| md | 16px | Toolbar section divider margins, status bar divider margins |
| lg | 16px | Toolbar horizontal padding from window edges |
| xl | 24px | Empty state content padding |
| 2xl | 32px | Not used in Phase 1 |
| 3xl | 48px | Not used in Phase 1 |

Note: `md` and `lg` share the same pixel value (16px) in this phase. They are kept as separate semantic tokens -- `md` for internal spacing between sibling groups, `lg` for edge padding against container boundaries. Both resolve to 16px from the standard set.

Exceptions: Toolbar button height is 28px (not a spacing token -- it is a component dimension derived from the 40px toolbar minus 6px vertical padding).

## Typography

Two weights only: 400 (regular) and 600 (semibold). Body and label text share the same size (13px) and are differentiated by weight.

| Role | Size | Weight | Line Height | Usage in Phase 1 |
|------|------|--------|-------------|-------------------|
| Body | 13px | 400 (regular) | 1.4 | Status bar text, tooltips, empty state body |
| Label | 13px | 600 (semibold) | 1.4 | Toolbar button labels, page counter, zoom percentage |
| Heading | 16px | 600 (semibold) | 1.2 | Empty state heading |
| Display | Not used | - | - | - |

**Font stack:** `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`

**Why 13px body, not 14 or 16:** Professional CAD/engineering tools use 12-13px for UI chrome to maximize canvas real estate. Bluebeam uses 12px, AutoCAD uses 13px. At 150% Windows DPI scaling, 13px CSS renders as ~20 physical pixels, which is readable. The canvas is the product, not the chrome.

**Why unify body and label at 13px:** A 1px difference between 12px and 13px is visually indistinguishable, especially in a dark UI. Using a single size with weight differentiation (regular vs semibold) creates clearer hierarchy while reducing the type scale to its minimum effective size.

## Color

This phase uses a dark theme consistent with professional engineering/CAD software.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | Canvas background (behind PDF), app background |
| Secondary (30%) | `#252526` | Toolbar, status bar, empty state card |
| Accent (10%) | `#0078d4` | Open PDF button (primary CTA), active page indicator, zoom percentage when non-100% |
| Destructive | Not used in Phase 1 | - |
| Surface border | `#3c3c3c` | Toolbar bottom border, status bar top border, toolbar section dividers |
| Text primary | `#cccccc` | All body text, labels, page counter |
| Text secondary | `#808080` | Disabled navigation buttons, secondary status info |
| Text on accent | `#ffffff` | Text inside accent-colored buttons |
| Hover surface | `#2d2d30` | Toolbar button hover state |
| Active surface | `#37373d` | Toolbar button active/pressed state |

Accent reserved for: the "Open PDF" primary CTA button background, the active page number highlight in the navigation cluster, and the zoom percentage text when zoom is not 100% (to signal non-default zoom state).

## Component Inventory

### Toolbar Button (icon + label variant)

```
[icon] Label
```

- Height: 28px
- Padding: 4px 8px
- Background: transparent (default), `#2d2d30` (hover), `#37373d` (active)
- Border-radius: 4px
- Icon size: 16x16px (Lucide default)
- Icon-to-label gap: 4px
- Text: 13px semibold (600) `#cccccc`
- Disabled state: opacity 0.4, cursor not-allowed
- Cursor: pointer (enabled), not-allowed (disabled)

### Toolbar Button (icon-only variant)

- Width: 28px, Height: 28px
- Padding: 6px
- Same background/hover/active/disabled as above
- Tooltip on hover (native `title` attribute -- no custom tooltip in Phase 1)

### Open PDF Button (primary CTA)

- Same dimensions as toolbar button (icon + label)
- Background: `#0078d4` (default), `#1a86db` (hover), `#0067b8` (active)
- Text: 13px semibold (600) `#ffffff`
- Icon: `FileUp` from Lucide, 16x16px, `#ffffff`

### Page Navigation Cluster

```
[<] Page 3 of 12 [>]
```

- Previous/Next: icon-only toolbar buttons using `ChevronLeft` / `ChevronRight`
- Page indicator: "Page {n} of {total}" in 13px semibold (600) `#cccccc`
- Previous disabled when page === 1; Next disabled when page === total
- Cluster centered horizontally in toolbar

### Zoom Controls

```
[Zoom Out] 125% [Zoom In] [Fit]
```

- Zoom In/Out: icon-only toolbar buttons using `ZoomIn` / `ZoomOut`
- Zoom percentage: 13px semibold (600), `#cccccc` at 100%, `#0078d4` at any other value
- Fit button: icon-only toolbar button using `Maximize` icon
- Zoom step: 25% per click (25%, 50%, 75%, 100%, 125%, 150%, 200%, 300%, 400%, 800%)
- Scroll wheel zoom: zoom toward cursor position, same step increments
- Zoom Out disabled at 25%; Zoom In disabled at 800%
- Positioned at right side of toolbar

### Status Bar

- Height: 28px
- Background: `#252526`
- Border-top: 1px solid `#3c3c3c`
- Three sections separated by `#3c3c3c` vertical dividers (1px wide, 16px tall, vertically centered)
- Left section: filename (truncated with ellipsis if > 40 chars), 13px regular (400) `#cccccc`
- Center section: "Page {n} of {total}", 13px regular (400) `#cccccc`
- Right section: "Zoom: {pct}%", 13px regular (400) `#cccccc`
- Horizontal padding: 16px from edges, 16px around dividers
- When no PDF loaded: all sections show em-dash `--`

### Canvas Viewport

- Fills remaining vertical space below toolbar, above status bar
- Background: `#1a1a1a` (always visible as a "desk" around the PDF page)
- PDF page rendered centered in viewport with padding
- Pan: middle-mouse-button drag or spacebar+left-click drag (standard CAD convention)
- Zoom: Ctrl+scroll wheel (toward cursor), or toolbar buttons
- Cursor: default (no PDF), grab (hovering canvas with PDF), grabbing (panning)
- No scrollbars -- pan is infinite canvas style

### Empty State (no PDF loaded)

Centered vertically and horizontally in the canvas viewport area.

```
        [FileUp icon, 48px, #808080]

       Open a PDF floor plan to begin

      Drag and drop a file here, or use
         the Open PDF button above.
```

- Icon: `FileUp` from Lucide, 48x48px, `#808080`
- Heading: "Open a PDF floor plan to begin" -- 16px semibold (600) `#cccccc`
- Body: "Drag and drop a file here, or use the Open PDF button above." -- 13px regular (400) `#808080`
- Gap between icon and heading: 16px
- Gap between heading and body: 8px
- The entire empty state area is a drop zone (drag-and-drop PDF files)
- Drop zone visual: dashed 2px `#3c3c3c` border rounded 8px, transitions to `#0078d4` border on drag-over

## Interaction Contracts

### File Open Flow

1. User clicks "Open PDF" button OR drags a PDF file onto the canvas viewport
2. If button: Electron `dialog.showOpenDialog` with filter `{ name: 'PDF Files', extensions: ['pdf'] }`
3. PDF loads; first page renders centered at fit-to-window zoom
4. Toolbar updates: page counter shows "Page 1 of {N}", zoom shows computed fit percentage
5. Status bar updates: filename, page, zoom

### Page Navigation

1. User clicks Next (or presses Right Arrow / Page Down)
2. Current page fades out (no animation in v1 -- instant swap)
3. Next page renders at the SAME zoom level and pan position (if the page is the same size), or fit-to-window if page dimensions differ
4. Page counter updates

### Zoom

1. User scrolls mouse wheel with Ctrl held
2. Zoom increments toward cursor position (the point under the cursor stays fixed)
3. Zoom percentage updates in toolbar and status bar
4. "Fit" button resets to fit-to-window zoom, centered

### Pan

1. User holds middle mouse button (or spacebar + left click) and drags
2. Canvas translates 1:1 with mouse movement
3. No momentum / inertia -- stops when mouse stops

### Keyboard Shortcuts (Phase 1)

| Shortcut | Action |
|----------|--------|
| Ctrl+O | Open PDF file |
| Left Arrow / Page Up | Previous page |
| Right Arrow / Page Down | Next page |
| Ctrl+0 | Fit to window |
| Ctrl+= | Zoom in |
| Ctrl+- | Zoom out |
| Space+drag | Pan |
| Ctrl+scroll | Zoom toward cursor |

## Accessibility Notes (Phase 1 Baseline)

- All toolbar buttons have `aria-label` attributes matching their tooltip text
- Disabled buttons use `aria-disabled="true"` and `tabindex="-1"`
- Keyboard focus ring: 2px solid `#0078d4` with 2px offset, visible on tab navigation
- File drop zone announces "File drop zone" to screen readers
- Page navigation announces "Page {n} of {total}" to screen readers via `aria-live="polite"` region

---

## HiDPI / Display Scaling

- Canvas dimensions are set to `cssWidth * window.devicePixelRatio` by `cssHeight * window.devicePixelRatio`
- CSS canvas dimensions remain at logical pixel size
- PDF.js render scale is multiplied by `devicePixelRatio`
- All toolbar/status bar dimensions are in CSS pixels (Electron handles DPI scaling for HTML elements)
- Tested target: 150% Windows display scaling (the most common non-100% setting on construction office monitors)
