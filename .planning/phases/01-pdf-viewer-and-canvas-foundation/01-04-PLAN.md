**File 1: src/renderer/src/hooks/usePdfRenderer.ts** -- Rewrite to add canvas caching and pre-rendering.

Keep the existing `clampedRenderScale` export unchanged (other tests depend on it).

Replace the `usePdfRenderer` hook internals with this approach:

1. **Module-level canvas cache:** Create a `Map<string, { canvas: HTMLCanvasElement; width: number; height: number }>` at module scope (keyed by `${documentId}-${pageNumber}` where documentId is the pdfDocument fingerprints string or a counter). This survives re-renders. Also store a `documentId` ref that resets the cache when a new PDF is loaded.

2. **Cache-first rendering:** In the effect that fires on `[pdfDocument, currentPage]`:
   - Check cache first: if `cache.has(key)`, immediately set pageCanvas and pageSize from the cached entry -- no async work, no delay.
   - If not cached, render as before (getPage + render), then store result in cache before calling setPageCanvas/setPageSize.

3. **Background pre-rendering of adjacent pages:** After the current page finishes rendering (whether from cache or fresh), queue pre-renders for pages `currentPage - 1` and `currentPage + 1` (if they exist and are not already cached). Use `requestIdleCallback` (or `setTimeout(fn, 0)` as fallback) so pre-rendering doesn't block the current page display. Pre-rendered results go into the cache but do NOT update the React state (they are for future navigation).

4. **Cache invalidation:** When `pdfDocument` changes (new file opened), clear the entire cache map. Use a ref to track the current document identity. Compare pdfDocument reference equality.

5. **Keep the return type identical:** `{ pageCanvas, pageSize }` -- no breaking change to CanvasViewport.

6. **Cancel in-flight pre-renders** when the document changes or the component unmounts. Use the existing `renderTaskRef` pattern and add a `preRenderTaskRefs` Map for background renders.

**File 2: src/renderer/src/components/CanvasViewport.tsx** -- Eliminate the blank flash.

1. **Remove the null return guard on line 123.** Replace `if (!pageCanvas || !pageSize) return null` with a pattern that keeps displaying the previous content:
   - Add a `useRef` to store the last valid `{ pageCanvas, pageSize }` pair.
   - Update the ref whenever `pageCanvas` and `pageSize` are both non-null.
   - Use the ref values for rendering: `const displayCanvas = pageCanvas ?? lastValidRef.current?.pageCanvas` and same for pageSize.
   - Only return null if there has NEVER been a valid render (truly initial state before any PDF is loaded). This is fine because EmptyState handles the no-PDF case.

2. **The Stage/Layer/KonvaImage** should use `displayCanvas` and `displayPageSize` instead of `pageCanvas` and `pageSize` directly.

3. The viewport-apply effect (lines 59-78) should still trigger on the real `pageSize` changes (when the actual new page is ready), not the stale display values. This ensures the zoom/pan snaps to the correct state once the new page arrives.

**Do NOT change:**
- The `clampedRenderScale` export function signature
- The `getCanvasControls` module-level pattern
- The `useViewportControls` or `useKeyboardShortcuts` integration
- Any other files
