## Critical Anti-Patterns (do NOT repeat these)

- **Sticky boolean flag for click-vs-drag disambiguation.** Once set true at peak max-travel, it cannot recover even if the gesture returns to its origin. Always compare FINAL mousedown→mouseup displacement instead. The principle generalizes beyond markup tools — any time you need "is this a click or a drag?", measure the resulting displacement, not the path. Mitigated structurally in commit a8c37eb (markupDraggedRef removed entirely).
- **Threshold-tuning before semantic-correction.** When a gate-too-tight bug surfaces, the cheap fix is to loosen the threshold. But if the gate is asking the wrong question, no threshold value will resolve it — you'll just shift where the bug bites. Always check whether the gate's semantics match user intent before adjusting its parameters.
